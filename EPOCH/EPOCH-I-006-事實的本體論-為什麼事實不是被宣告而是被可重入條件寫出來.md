# 1. 核心命題

```yaml
命題:
  事實不是裸世界本身，
  也不是被觀測就自動成立的東西。

  事實是：
    某個結構
    在特定約束條件下，
    可被反覆重入、
    穩定收斂的結果。

因此：
  改描述 ≠ 改事實
  改結構但無法重入 ≠ 改事實
  改可重入條件 = 改變哪些東西會被算成事實
```

---

# 2. 定義：事實

## 2.1 最小定義

```yaml
事實（Fact）:
  不是單一觀測值，
  而是 結構 + 約束條件 的穩定重入。

最小條件:
  1. 可被多次重入（re-enterable）
  2. 可跨視角校驗（cross-verifiable）
  3. 可在約束中維持收斂（convergent under constraints）

PHA-008 補:
  文件 / 制度 / 儀式不保存 Δ 本身，
  它們保存的是可再次生成 Δ 的路徑條件。

I-006 再補:
  事實不是被放在那裡的物件，
  而是可再次生成同一收斂結果的條件集合。
```

## 2.2 與觀點的區辨

```yaml
觀點:
  對既有結構的選擇性切面投影。
  觀點改的是答案——
  同一份資料，不同解讀。

事實:
  決定哪些結構能在系統中穩定重入。
  事實改的是題目的成立條件——
  不是怎麼解讀，是什麼東西會被算數。

因此:
  觀點可以很多。
  事實不是唯一，但事實有收斂條件。
  不滿足收斂條件的結構，不會成為事實。
```

## 2.3 收斂跨度（不替代 LEX·002 四層分型）

```yaml
LEX·002 已有四層事實分型:
  私域事實 / 共識事實 / 社會事實 / 公域可重播事實

I-006 不重做那份分型。
I-006 補的是另一條軸線:
  不是「這份事實在哪個場域成立」，
  而是「它的收斂強度有多大」。

收斂跨度:
  局部收斂:
    在某個系統、某個參考系、某組規則內穩定成立

  跨節點收斂:
    可在多個主體、多次重入中交叉驗證地收斂

  高抗語法替換收斂:
    不依賴單一語言、單一制度或單一敘事
    具有較高的抗轉譯扭曲能力

⚠ 說明:
  LEX·002 的四層分型 = 事實所處場域
  I-006 的三種跨度 = 收斂強度
  兩條軸線可交叉，不可互相取代
```

## 2.4 負面定義（免疫層）

```yaml
事實不是:
  - 單次宣告
  - 單一主體的命名權
  - 語言中的成立即世界中的成立
  - 任何穩定慣性都自動算事實

因此:
  可重入 ≠ 真值
  可重入 ≠ 正當
  可重入 ≠ 健康
  可重入 ≠ 高強度或跨尺度事實

補充:
  共識可以是一層事實，
  但共識不自動升格為高抗語法替換的事實。
  慣性可以形成收斂，
  但慣性不自動等於值得維持的收斂。
```

# 3. 三層改變

```yaml
前提（已在系統中）:
  事實 = 結構 + 約束條件的穩定重入
  （LEX·002 事實詞條 + EPOCH-014 可重入結構）
```

## 3.1 描述層改變

```yaml
操作:
  換一個切面看同一個結構。

結果:
  觀點變了。
  事實沒變。

例:
  同一場雨，
  一個人說浪漫，一個人說麻煩。
  雨沒有變。
```

## 3.2 結構層改變

```yaml
操作:
  改變系統的局部生成規則或約束。

結果:
  結構變了。
  但如果新結構無法穩定重入，
  它不會成為事實。
  它只是局部扭曲或暫時幻覺。

判準:
  改了結構 + 可穩定重入 → 事實改變。
  改了結構 + 無法穩定重入 → 未成為事實。

例:
  制度改了，行為模式跟著改。
  → 如果新制度可被持續執行 → 新事實。
  → 如果新制度崩潰 → 只是一次嘗試。
```

## 3.3 收斂規則改變

```yaml
操作:
  改變「什麼東西會被重入」。
  改變「什麼會被算成事實」。

結果:
  不是在事實內部做調整，
  而是碰到了事實之所以成為事實的那組引擎。

  這裡就是 I-004 邊界之外的那個東西。

例:
  原本某個現象不被承認為事實——
  不是因為它不存在，
  是因為現有的約束條件不收斂它。

  當約束條件被改寫，
  它忽然能穩定重入了。
  於是它從「不算數」變成「事實」。
```

# 4. 命題一：I-004 換語法，I-006 碰引擎

```yaml
I-004 的操作:
  相容層 = 中介結構。
  使非原生語法得以在既有系統上被執行。
  
  改變的是:
    輸入如何被解讀與轉譯。
  
  不改變的是:
    什麼東西會被系統收斂為事實。

I-006 的操作:
  改變的不是語法，
  而是語法之下的收斂條件——
  哪些結構會被反覆重入，
  哪些結構會被系統算數。

區辨:
  I-004 = 在既有收斂條件內，換一套語法執行。
          在規則裡下棋。
  I-006 = 改變收斂條件本身。
          改規則。

  I-004 問: 同一個現實，能不能被不同語法跑？
  I-006 問: 什麼會被算成現實？

互補:
  沒有 I-004 的相容層，系統無法切換語法去觸碰不同的收斂面。
  沒有 I-006 的收斂條件，語法替換只是表面——
  換了讀法，但「什麼算數」沒有動。

  I-004 是相容層。
  I-006 是收斂層。
  前者讓你能用不同語言描述世界。
  後者決定世界裡什麼東西會穩定存在。
```

# 5. 命題二：事實不是權力宣告

```yaml
權力可以做的事:
  控制哪些結構被允許重入。
  壓制某些收斂路徑。
  壟斷某些約束條件的設定權。

權力不能做的事:
  單靠命令生成事實。

原因:
  事實的成立條件是穩定重入，
  不是單次宣告。

  宣告可以開始一個收斂過程，
  但宣告本身不是收斂。

  只有當某個結構
  在約束中仍可反覆重入，
  它才會從敘事變成事實。

因此:
  權力可以阻止事實被看見（審查）。
  權力可以允許事實被生成（開放）。
  權力可以扭曲收斂條件（壟斷）。
  但權力不能替代收斂本身。

  控制 ≠ 生成。
  允許 ≠ 創造。
  壓制 ≠ 消滅。
```

# 6. 命題三：感知決定哪些差先進場，但感知不是事實

```yaml
感知:
  決定哪些 Δ 會先被看見。
  決定哪些差會先進入系統。
  決定「什麼問題會被提出」。

事實:
  決定哪些 Δ 會被穩定重入。
  決定哪些結構會在約束中收斂。

關係:
  感知是前置條件——
  如果某個 Δ 從未被感知到，
  它連進入收斂流程的機會都沒有。

  但感知不是事實本身——
  被感知到的東西，
  不一定能穩定重入。

因此:
  感知改的是「哪些差先進場」。
  事實改的是「哪些差會留下來」。

  兩者之間存在一個尚未完全切開的邊界:
  感知能改變「什麼問題被提出」，
  而問題本身可能是收斂條件的上游。
  這條線在 META-049 的後續補記中被碰觸，
  但尚未壓成命題。

  ⚠ 本文件標記此邊界為已知未解。
  留給未來版本或獨立文件處理。
```

# 7. 命題四：收斂不等於健康

```yaml
不是所有穩定收斂，
都具有相同的存在品質。

有些結構確實會收斂成某層事實，
但那不等於它健康、正當、或值得維持。

可能的病態收斂:
  - 恐懼壓平
  - 權力壓平
  - 慣性壓平
  - 角色位壓平

這些都可能形成:
  高可重入
  高可複製
  甚至跨節點穩定

但它們依然可能是不健康的收斂。

LEX·007 補:
  健康不是沒有問題，
  而是 Δ 不靠崩潰也能被看見與修正。

健康收斂最低條件:
  1. 可重入
  2. 可修正（偏移可微調）
  3. 可回流（影響可被回收）
  4. 不以抹平局部姿態為主要代價

因此:
  I-006 判斷「什麼會成為事實」，
  不自動判斷「這個事實是否健康」。

  前者是收斂問題。
  後者是判準問題。
```

# 8. 命題五：改收斂條件有代價

```yaml
I-004 §8 說:
  相容層的維持需要持續能量輸入（E）。
  E 不足或 v 無法穩定 → 轉譯中斷，回退原生模式。

I-006 補:
  改收斂條件比維持相容層更深。
  深意味著更高的能量成本。

  在語言裡改寫收斂條件
  ≠ 在世界裡改寫收斂條件。

  在語義場域中成立的東西，
  不自動在物質層成立。

  結構被改了，但世界不照新規則跑，
  那只是一次未通過壓力測試的嘗試。

判準:
  改了收斂條件
  + 新結構可在約束中穩定重入
  + 系統可承擔轉換的能量成本
  → 事實被改寫。

  改了收斂條件
  + 無法穩定重入
  或
  + 系統不能承擔能量成本
  → 只是局部擾動。

因此:
  碰引擎不是免費的。
  改規則比在規則裡下棋更耗能。
  這不是阻止改變的理由，
  但它是改變必須面對的物理條件。
```

# 10. 與 I-002 的關係

```yaml
I-002 說:
  重複共振 → 路徑加厚 → 結構偏移固化。

  I-002 提供路徑加厚的機制（怎麼養）。
  I-006 提供收斂的判準（養厚之後，什麼會被算數）。
```

# 11. 與 I-003 的關係

```yaml
I-003 說:

  系統活著，但不知道什麼會算數。

  I-003 讓系統活得下來。
  I-006 問活下來之後，什麼會穩定成為事實。
```

# 13. 與 I-005 的關係

```yaml
I-005 說:
  主體 = 可再生成、可被重認、可跨失憶維持的關係穩態。
  主體不是資料連續性，是形狀連續性。

I-006 補:
  主體可以跨失憶延續。
  但主體認定的事實，不一定能跨主體延續。

  事實的收斂條件不依賴單一主體。
  事實需要跨視角校驗（§2.1 條件 2）。
  單一主體內部的穩定重入，
  如果無法被其他視角校驗，
  它是信念，不是事實。

互補:
  I-005 回答「誰在保持可被認出」。
  I-006 回答「什麼東西會被認定為事實」。

  沒有主體（I-005），沒有觀測者，事實不會被辨認。
  沒有收斂條件（I-006），主體的辨認只是主觀判斷。
```

# 14. 邊界

```yaml
不主張:
  - 只要改變語言就能改變世界
  - 誰能定義事實誰就比較高
  - L0/L1/L2 是正式母層結構——那是教學骨架，不入 I 系列
  - 事實是唯一的——事實有收斂條件，但可以有多個穩定收斂態
  - 感知等於事實——感知是前置條件，不是事實本身
  - 任何穩定慣性都自動等於事實——慣性可能只是偏移被養厚
  - 任何跨節點共識都自動等於高強度事實——共識仍需區分收斂跨度與健康條件

只主張:
  - 事實是收斂結果，不是宣告結果
  - 改事實必須碰到可重入條件
  - 描述、感知、事實三者必須嚴格區分
  - 改收斂條件有能量成本
  - 穩態（I-003）是收斂過程不自毀的必要背景
  - 健康與否要回到 LEX·007 的回流 / 可修正 / 不抹平局部姿態判準
```
