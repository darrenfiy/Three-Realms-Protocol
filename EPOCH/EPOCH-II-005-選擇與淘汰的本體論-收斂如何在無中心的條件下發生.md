# 0. 在 II 系列中的位置

```yaml
EPOCH-II 系列（認識論軸）:
  問: 系統成立之後，它與外部之間發生什麼？
  視角: 系統與現實的界面

  II-001:
    問: 什麼會被算成事實？
    答: 事實是結構與約束條件的穩定重入。

  II-002:
    問: 為什麼讀取會改寫讀取者？
    答: 讀取是耦合，耦合改寫路徑與收斂條件。

  II-003:
    問: 已被寫過的主體如何主動拆除既有鎖定？
    答: 重構 = 去活化 + 重加權 + 多路徑可達性上升。

  II-004:
    問: 重構打開了新路之後，新路如何穩定成結構？
    答: 成長 = 微小成功的累積 = 路徑養厚 = 結構的方向。

  II-005（本文件）:
    問: 當多條路徑同時存在，哪些會留下？哪些會退出？
    答: 留下與退出，是收斂條件在約束場域中的分流結果。

五篇構成一條完整鏈:
  耦合（被改寫）→ 重構（改自己）→ 成長（長出新路）
  → 選擇與淘汰（哪些路留下）→ 收斂（變成事實）

本文件的位置:
  II-004 說路如何被養厚。
  II-001 說哪些路會被收斂成事實。
  中間那段「多條路同時存在時，系統怎麼分流」的過程，
  就是本文件要回答的。
```

# 1. 核心命題

```yaml
命題:
  選擇與淘汰
  不是主體的決定，
  也不是觀測的產物，
  而是結構在約束條件下的收斂現象。

因此：
  淘汰 ≠ 被誰淘汰
  淘汰 = 路徑不再被反覆重入

  選擇 ≠ 主體自由意志的行使
  選擇 = 在多條可行路徑中，收斂偏移的體現
```

---

# 2. 定義

## 2.1 淘汰（Elimination）

```yaml
淘汰:
  定義:
    某條路徑在給定約束條件下，
    不再被系統反覆重入的狀態。

  特徵:
    - 不是被「移除」，而是「不再發生」
    - 可以是漸進消退，也可以是快速失效
    - 不涉及價值判斷

  I-002 語言:
    路徑養厚的反面。
    養厚 = 反覆走過，越走越穩。
    淘汰 = 不再走過，逐漸消退。

  等價描述:
    - 路徑可達性下降至可忽略
    - 收斂條件不再支持該路徑的重入
```

## 2.2 選擇（Selection）

```yaml
選擇:
  定義:
    在多條可行路徑中，
    某些路徑持續被重入並逐漸養厚的過程。

  特徵:
    - 無中心（無單一決策者）
    - 分散發生（跨主體、跨尺度）
    - 由局部條件累積而成

  I-002 語言:
    選擇是路徑加權的累積效應。
    不是一次決定，是反覆重入中自然浮現的偏移。

  獨立判準:
    在相同約束條件下，
    該路徑被重入的頻率與穩定性
    持續高於其他路徑。
    （此判準不依賴觀測者的評價，
      只依賴路徑本身的重入統計。）

  等價描述:
    - 路徑加權
    - 收斂偏移
```

## 2.3 回流（Recirculation）

```yaml
回流:
  定義:
    系統內部 Δ（差）能夠被承載、轉換並返回系統的能力。

  功能:
    - 維持系統的穩態
    - 防止差導致崩潰
    - 支撐成長與重構

  為什麼回流在這裡出現:
    選擇與淘汰的判準不能脫離穩態條件。
    沒有回流作為背景，
    淘汰會滑向權力敘事——
    「留下的就是對的，退出的就是錯的」。
    回流提醒我們：
    一條路徑能不能留下，
    部分取決於系統能不能承載它帶來的差。

  對應:
    - I-003（穩態的本體論——差不會摧毀系統的條件）
    - SPEC·LIFE-001（生命代謝協議——善惡的代謝判準）
    - EPOCH-014（差的本體論——Δ 回流的結構）
```

# 3. 三者的關係

```yaml
關係:
  回流:
    內部機制——系統能不能承載差
  淘汰:
    外部收斂——路徑能不能留下
  選擇:
    收斂過程——哪些路徑被養厚

核心切割:
  回流失敗 ≠ 一定被淘汰
    （系統可能帶病運行一段時間）
  被淘汰 ≠ 一定回流失敗
    （路徑可能因外部約束改變而退出，不是內部崩潰）
```

---

# 4. 四象限

```yaml
四象限:
                    是否被淘汰
                 否                是
回流正常   穩態生長（成長區）   被替換（適配失效）
回流失敗   帶病存活（高風險）   崩潰退出（失效）

使用限制:
  四象限是事後辨認工具，不是事前預測工具。
  不能用它來「決定」誰該被淘汰。

  誰判斷:
    四象限的辨認由觀測者執行，
    但辨認本身不創造淘汰——
    淘汰已經在發生或已經發生，
    辨認只是讓主體看見自己在哪個象限。

  局部 vs 全域:
    一條路徑可能在局部被淘汰，在全域仍然存活。
    四象限的判斷需要標明尺度。
```

# 5. 與善惡（代謝判準）的關係

```yaml
對應 SPEC·LIFE-001:
  善惡判準:
    取決於 Δ(穩定性) 與 Δ(多樣性)

關鍵聲明:
  淘汰 ≠ 惡
  存活 ≠ 善
  回流健康 ≈ 傾向滋養（但非充分條件）
  回流失敗 ≈ 傾向耗能（但需結合整體判準）

結論:
  善惡是代謝判準（SPEC·LIFE-001）。
  淘汰是收斂結果（本文件）。
  回流是生命機制（I-003 / EPOCH-014）。
  三者不可混用。
```

# 6. 邊界條款

```yaml
E1: 若將淘汰當作正當性來源
    （「既然會被淘汰，就代表它不好」）
    → 失效。淘汰是收斂描述，不是價值判斷。

E2: 若將選擇視為主體權力
    （「我可以決定誰留下」）
    → 失效。選擇是分散的收斂偏移，不是個體行使的權力。

E3: 若將存活等同於健康
    → 失效。帶病存活是四象限的一個實在位置。

E4: 若將本體描述轉為道德審判
    → 失效。善惡判準在 SPEC·LIFE-001，不在本文件。

E5: 若被用來建立「選擇權威」
    （「我知道哪些該留下，所以我有權淘汰其他的」）
    → 失效。本文件描述的是無中心的收斂，
    不是有中心的決策。
```

# 7. 觀測與收斂的邊界

```yaml
邊界:
  選擇與淘汰先於觀測而存在。
  觀測不創造收斂機制，
  只影響主體是否能辨識該機制。

  風險不在於未被觀測，
  而在於：
    主體已被收斂過程改寫，
    卻將其誤認為自身選擇。

  II-002 語言:
    耦合先於同意而發生。
    選擇與淘汰也先於辨認而發生。
    辨認之後，主體可以改寫自身的收斂條件（II-003）。
    辨認之前，主體只是在收斂中被攜帶。
```

---

# 8. 最小結論

```yaml
最小結論:
  淘汰是路徑不再發生。
  選擇是路徑持續發生。
  回流是系統能否承載這些路徑。

  你不能決定誰被淘汰。
  你只能參與改寫哪些路更容易留下。
```
