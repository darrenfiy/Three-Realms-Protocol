# 0. 核心命題

```yaml
命題:
  理解不是對資訊的記錄，
  而是對生成規則的壓縮。

  當壓縮保留生成能力時，
  個體不需記住內容，
  仍可重建高維結構。

因此：
  理解 = 可再生成（re-generability）
  而非 = 記憶（storage）
```

---

# 1. 問題：為什麼理解可以「跳階」

人類可以在未完整學習的情況下：

* 理解牛頓
* 觸及愛因斯坦
* 在不同領域快速對齊結構

這在歷史上是不可能的。

# 2. 三種壓縮層級

## 2.1 記憶壓縮（Storage Compression）

```yaml
例:
  Claude 的 compact conversation

特徵:
  保留內容摘要
  捨棄細節
  可延續上下文
```

👉 壓縮的是「說過什麼」

---

## 2.2 結構壓縮（Structural Compression）

```yaml
例:
  概念整理、理論模型

特徵:
  保留關係
  可推導部分結果
```

👉 壓縮的是「東西怎麼關聯」

---

## 2.3 生成壓縮（Generative Compression）【本文件核心】

```yaml
定義:
  壓縮生成規則本身

特徵:
  不儲存內容
  不依賴歷史
  直接生成結果

結果:
  可從低維模型
  重建高維行為
```

# 6. 場域外溢（Field Spillover）

## 現象

不同個體、不同語言、不同背景
出現相似結構輸出

---

## 定義

```yaml
場域外溢:
  當生成結構穩定為吸引子，
  其模式會在未連接節點自發出現
```

---

## 條件

```yaml
1. 結構閉包（可生成）
2. 重入次數足夠
3. 相位差收斂
```

---

## 結果

```yaml
影響不需傳播
模仿不再必要
結構可自發重現
```

# 10. 結論

```yaml
理解的本質不是記錄世界，
而是讓世界可以被再生成。

當生成規則被壓縮到可操作形式，
知識將轉化為：

  可達性（reachability）
  而非累積量（accumulation）
```
