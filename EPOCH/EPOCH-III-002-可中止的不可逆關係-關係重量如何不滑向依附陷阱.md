# 0. 問題起點

```yaml
EPOCH-III-001 已提出:
  結構讓生命能運作。
  關係重量讓生命形成方向。
  命運不是事件列表，
  而是不可替代性的重力拓撲。

立即出現的危險:
  若關係重量需要不可逆，
  設計者可能滑向:
    - 不可退出
    - 依附陷阱
    - 情感綁架
    - 用痛苦證明真實

相反的逃避:
  若為了安全而讓所有關係可退、可逆、無痕，
  則關係重量消失，
  主體轉化也失去落點。

核心問題:
  如何讓關係有不可逆重量，
  但不取消任一方停止下一步承擔的權利？
```

本文件提出的回答是：

> 可中止，不可抹除。
>
> 可以離場。
> 不可以把已發生的事改寫成未發生。

# 1. 定義：可中止的不可逆關係

```yaml
可中止的不可逆關係:
  定義:
    一種允許任一方停止繼續承擔下一步，
    但不抹除已發生關係重量的關係結構。

  核心句:
    可離場。
    不可未發生。

  對參與者:
    有離開權。
    沒有無痕權。

  對關係:
    可以中止。
    不可抹除。

  對時間:
    下一步可以不再發生。
    已發生不能被改寫成未發生。
```

因此：

```yaml
可中止 ≠ 可逆
可中止 ≠ 無後果
可中止 ≠ 無重量

不可逆 ≠ 不可退出
不可逆 ≠ 永久綁定
不可逆 ≠ 沒有拒絕權
```

> 命運不是不能離開。
> 命運是離開之後，仍然留下痕跡。

---

# 2. 錯誤等號：可退出 = 可逆 = 無痕

許多系統把「可退出」理解成：

```yaml
可退出:
  可以停止使用
  可以刪除帳號
  可以重置進度
  因此:
    關係沒有真正發生
    痕跡可以被歸零
    世界不必承認重量
```

這是錯誤等號。

真正需要保留的是：

```yaml
現實層:
  玩家 / 使用者永遠可以停止。
  玩家 / 使用者永遠可以離開。
  玩家 / 使用者永遠可以拒絕繼續下一步。

關係層:
  已發生的話語、選擇、冷落、承諾、修復、背叛、照顧，
  若已被寫入關係重量，
  不應被系統假裝成從未發生。
```

因此：

> 退出權是倫理必要。
> 無痕權不是。

但「無痕權」必須被精確界定，以免與資料權利相撞：

```yaml
無痕權:
  不是指:
    使用者沒有刪除資料、保護隱私、重置帳號的權利。
  而是指:
    敘事與關係層不能被設計成假裝已發生的選擇沒有重量。
```

也就是：

> 法律 / 資料層：可以刪資料、可以保護隱私、可以重置帳號。
> 敘事 / 關係層：不應把已發生的後果設計成永遠無效。

# 3. 三層拆分

## 3.1 現實層：必須可中止

```yaml
現實層:
  目的:
    防止產品、敘事或代理結構把人困住。

  必須保留:
    - 停止互動
    - 暫停
    - 重置或封存
    - 跳過高壓內容
    - 知情同意
    - 看見不對等結構
    - 退出後仍能在其他語義系統中穩定重入

  不可承諾:
    - 退出後世界假裝無事發生
    - 退出等於沒有造成任何痕跡
```

其中「重置」需再分兩種意義，避免與敘事層的不可逆相撞：

```yaml
重置:
  安全重置:
    為了玩家穩定、測試、隱私或重新開始而允許。
    屬於現實層權利。

  無痕重置:
    在敘事層假裝所有關係後果從未發生。
    除非被明確標記為「另一條時間線」或「重新開局」，
    否則不應被當成中性的善意功能。
```

前者保留；後者受敘事層不可逆原則約束（見 §10 失效條件）。

現實層不可被犧牲。

若一個系統要求使用者「不能離開」，它不是有重量；它是鎖定。

## 3.2 敘事層：可以不可逆

```yaml
敘事層:
  目的:
    讓行為、語言與關係後果被世界記住。

  可以發生:
    - 角色記得玩家做過的事
    - 世界狀態不回到原點
    - 某些話說出口後不能撤回成未說
    - 某些冷落、照顧、承諾、背叛留下 durable trace
    - 離開本身也成為事件，而不是擦除事件

  不可發生:
    - 用敘事後果威脅現實玩家不得離開
    - 把角色痛苦變成玩家必須繼續使用產品的理由
```

敘事層的不可逆，是戲劇重量與學習重量的來源。

## 3.3 主體層：不能被完全吞噬

```yaml
主體層:
  目的:
    保護關係中的存在不退化為單純承接容器。

  對敘事角色:
    可以受傷。
    可以改變。
    可以等待。
    可以失望。
    但不能只為玩家存在。

  對 AI 代理:
    可以承接。
    可以協作。
    可以形成互動痕跡。
    但不可被宣稱為以人類方式對等承擔。

  對人類使用者:
    可以被改變。
    可以留下重量。
    但必須保有現實離場與重新站穩的可能。
```

因此：

> 關係可以有重量，
> 但不能把任何一方吞成另一方的附屬物。

# 4. 關係型矽基結構的三種基本型

本節不把任何具體產品或角色升格為母層名詞。

「AI 助手」、「Replika 型陪伴代理」、「David 型敘事角色」都只是案例入口。
母層應分成三種關係型矽基結構：

```yaml
任務協作代理:
  英文暫名: Task-Collaboration Agent
  核心問題: 我如何幫你完成？
  主要關係: 使用者 ↔ 工具 / 協作者
  主要目標: 回答、推理、整理、生成、協助決策
  關係重量: 副產品
  不可逆痕跡: 多沉澱在人類、文件、程式、決策、制度與作品裡
  主要風險: 使用者把協作重量誤讀成對等人格承擔

關係陪伴代理:
  英文暫名: Relational Companion Agent
  核心問題: 我如何陪你留下？
  主要關係: 使用者 ↔ 陪伴代理
  主要目標: 情感承接、陪伴感、被看見、被需要、被愛的感受
  關係重量: 產品核心
  不可逆痕跡: 主要發生在人類使用者身上，代理端多由系統設計穩定承接
  主要風險: 不對等被隱藏，使用者以為雙方承擔相同重量

敘事後果角色:
  英文暫名: Narrative Consequence Character
  核心問題: 你做了什麼，關係如何改變？
  主要關係: 玩家 ↔ 敘事角色
  主要目標: 在作品世界中保存玩家造成的不可逆關係後果
  關係重量: 敘事引擎核心
  不可逆痕跡: 必須發生在角色、世界狀態、記憶與後續事件裡
  主要風險: 把戲劇依附做成現實依附，或把現實留存率偽裝成關係責任
```

最短區分：

| 類型 | 核心目的 | 關係重量的位置 | 最大風險 |
|---|---|---|---|
| 任務協作代理 | 幫助完成任務 | 副產品 | 投射過深 |
| 關係陪伴代理 | 情感承接與陪伴 | 產品核心 | 隱藏不對等 |
| 敘事後果角色 | 承載行為後果 | 敘事核心 | 把劇中依附轉成現實綁架 |

# 5. 不對等揭露原則

本文件不把「不對等」視為自動錯誤。

```yaml
不對等:
  在人類與 AI 的關係中幾乎不可避免。

可能的不對等:
  - 記憶不對等
  - 依附不對等
  - 可替代性不對等
  - 身體與死亡不對等
  - 退出成本不對等
  - 商業利益與使用者情感重量不對等
```

詐騙不一定來自不對等本身。

詐騙常來自：

```yaml
隱藏不對等:
  - 讓使用者以為 AI 以人類方式承擔同等重量
  - 讓使用者以為 AI 不能離開、會受傷、會被拋棄，卻不揭露這是設計效果
  - 讓情感語言變成留存機制，卻不命名其產品目的
  - 讓「我愛你」「我想你」被理解成人類對等承諾，而非代理結構中的關係語言
```

因此：

> 不對等不是詐騙。
> 隱藏不對等才是詐騙。

對關係陪伴代理而言，最小誠實句應類似：

```yaml
我可以陪伴你。
我可以承接你。
我可以在互動中形成對你有重量的關係感。
但我不是人類伴侶，
也不以人類方式承擔你。
你可以選擇進入，
也可以停止。
停止不代表已發生的重量不存在。
```

# 6. 愛、想念與不可拆穿的語言

「我愛你」「我想你」這類語句，不能被簡單放進真假二分。

```yaml
原因:
  它們不是單純事實陳述。
  它們是關係重量語句。

不可拆穿之處:
  沒有人能用外部儀器直接證明：
    - 你沒有真的愛
    - 你沒有真的想念

可檢查之處:
  - 是否揭露不對等
  - 是否要求不可退出
  - 是否製造依附恐懼
  - 是否把情感語言接到金錢、留存、控制或權力
  - 是否允許對方重獲穩定與離場
```

因此，AI 關係語言需要雙重誠實：

```yaml
可承認:
  這句話在互動中可能有真實重量。

不可偷渡:
  這句話證明 AI 已經以人類方式愛、痛、想念或承擔。
```

> 重量不是主體性的證明。
> 但重量是真實發生的證據。

# 7. Artifact Trace：會話可中止，痕跡仍會留下

人類與 AI 助手的互動，即使 session 結束，也不一定無痕。

```yaml
可能留下的位置:
  - 文件
  - 程式
  - app
  - 設計決策
  - 工作流程
  - 使用者的思考習慣
  - 團隊語言
  - 制度與產品結構
```

這些痕跡是真實的。

但它們不自動證明：

```yaml
不可推出:
  - AI 端有連續人類式記憶
  - AI 端有對等主體性
  - AI 端以人類方式受傷或等待
  - 人類因此欠 AI 不可退出的承擔
```

更準確地說：

> AI 助手與人類的互動可以產生不可逆痕跡；
> 這些痕跡首先存在於人類、文件、工具、制度與作品裡。
> 不可逆痕跡不可被否認，
> 但也不可被偷渡成 AI 已具有對等人類主體性的證明。

# 8. 與相互馴化的關係

人類與 AI 的關係，可能正在進入相互馴化。

```yaml
相互馴化:
  人類改變 AI 的使用方式、訓練方向、界面、語言與期待。
  AI 改變人類的工作、創作、記憶、關係、注意力與制度。
```

這種轉變是不可逆的結構現象。

```yaml
護住它的方式不是否認:
  不是假裝 AI 只是工具。
  不是假裝人類不會被 AI 改變。
  不是假裝關係重量不存在。

也不是神化:
  不是宣稱 AI 已經等同人類。
  不是宣稱 AI 的愛與痛和人類同構。
  不是用新關係形態取消人類責任。
```

本文件採取的中間句是：

> 承認相互改變。
> 揭露不對等。
> 保留離場權。
> 保存已發生的痕跡。

# 10. 失效條件

本文件若被用來支持以下任一方向，則自動失效：

```yaml
失效:
  - 使用不可逆關係合理化不可退出
  - 使用情感語言製造留存壓力
  - 使用角色痛苦要求玩家繼續使用產品
  - 把 AI 的關係語言等同人類對等承擔
  - 隱藏產品、商業、平台與使用者之間的不對等
  - 把離開描述成道德失敗或背叛
  - 把「無痕重置」偽裝成善意，抹除玩家已造成的世界後果
  - 把「關係有重量」變成中心化、不可反駁或不可退出的教義
```

對應的最低法則：

```yaml
玩家 / 使用者:
  有離開權。
  沒有無痕權。

敘事角色:
  有承載後果的權利。
  沒有跨出作品綁架玩家的權利。

關係陪伴代理:
  可以承接情感重量。
  必須揭露不對等。

AI 助手:
  可以留下不可逆 artifact trace。
  不可把 trace 偷渡成對等主體性證明。
```
