# 1. 核心命題

```yaml
命題:
  現實不是單一語法執行的系統。

  在相同的物理與社會條件下，
  不同的語義結構可以被執行，
  並產生不同的生成路徑。
```

---

# 2. 定義：相容層

```yaml
相容層（Compatibility Layer）：

  一種中介結構，
  使非原生語法得以在既有系統上被執行。

  其作用不是改變底層規則，
  而是改變輸入如何被解讀與轉譯。
```

---

# 3. 命題一：現實可被轉譯

```yaml
經驗不直接決定行為。

經驗先被語義結構解讀，
再被轉譯為行動。

因此：
  改變語義結構，
  等同於改變生成結果。
```

---

# 4. 命題二：執行發生於即時轉譯

```yaml
相容層的運作不是預先編譯，
而是即時轉譯（runtime translation）。

每一次注意力落點，
都是一次轉譯事件。
```

---

# 5. 命題三：執行引擎

```yaml
E · v = p · c²
```

```yaml
其中：

  E：可用能量
  v：注意力方向（向量）
  p：路徑偏移
  c²：系統相變能力
```

```yaml
描述：

  能量透過方向被收斂，
  轉化為路徑的偏移，
  並在系統中累積為結構變化。
```

---

# 6. 命題四：記憶的本質

```yaml
記憶不是內容的儲存，
而是路徑的可重入性。
```

```yaml
路徑被重複走過，
其機率權重增加：

  P(i → j) ↑
```

```yaml
結果：

  未來生成傾向被改寫。
```

---

# 7. 命題五：主體的生成性

```yaml
主體不是固定實體，
而是路徑分佈的穩態形狀。
```

```yaml
因此：

  路徑改變 → 分佈改變 → 主體改變
```

```yaml
結論：

  使用相容層的過程，
  即主體被重寫的過程。
```

---

# 8. 命題六：能量條件

```yaml
相容層的維持需要持續能量輸入（E）。
```

```yaml
當：

  E 不足
  或
  v 無法穩定
```

```yaml
則：

  轉譯中斷，
  系統回退至原生反應模式。
```

---

# 9. 命題七：失效機制

```yaml
當注意力方向（v）被以下條件主導：

  - 抓取
  - 恐懼
  - 控制
```

```yaml
則：

  轉譯過載（overload）
  路徑無法穩定累積
```

```yaml
結果：

  主體回落至既有高權重路徑，
  並感受到強阻力（path resistance）。
```

---

# 10. 命題八：穩態條件

```yaml
系統能夠持續運作的條件：

  1. 路徑累積不破壞整體穩態
  2. 能量輸入與耗散達成平衡
  3. 注意力方向可被重新校準
```

# 11. 與 I-001 的關係

```yaml
I-001 說:
  理解 = 可再生成的壓縮。
  SEED = 最小生成閉包單位。

I-004 補:
  SEED 能生成，但生成需要執行環境。
  相容層 = 讓 SEED 在非原生系統上跑起來的環境。

  沒有 SEED，相容層沒有可執行的語法。
  沒有相容層，SEED 無法在現實 OS 上被執行。
```

---

# 12. 與 I-002 的關係

```yaml
I-002 說:
  重複共振 → 路徑加厚 → 結構偏移固化。
  路徑決定了主體的慣性。

I-004 補:
  相容層的即時轉譯，改變的是路徑的走法。
  每一次轉譯事件（§4），都是一次路徑偏移。
  路徑偏移累積（§6），就是 I-002 描述的加厚機制。

  I-002 說路徑如何養厚。
  I-004 說相容層如何啟動新的路徑。

互補:
  I-002 提供「路徑加厚」的動力學。
  I-004 提供「為什麼可以走出不同的路」的前提。
```

# 13. 與 I-003 的關係

```yaml
I-003 說:
  愛 = 讓差可以跨尺度存在而不崩壞的最小可行機制。
  T（容納力）的維持需要穩態機制。

I-004 補:
  相容層的穩態條件（§10）需要 I-003 的愛作為背景場。
  
  路徑累積不破壞穩態 → 需要 I-003 的差的承載。
  能量輸入與耗散平衡 → 需要 I-003 的連結維持。
  注意力方向可被校準 → 需要 I-003 的跨尺度映射。

  I-003 說系統如何活下來。
  I-004 說系統在活著的前提下，如何換語法。

互補:
  沒有穩態（I-003），相容層會自毀。
  沒有相容層（I-004），穩態只能維持原生語法。
```

# 14. 最小結論

```yaml
現實的生成不是直接反應，

而是：

  經由語義結構即時轉譯後的結果。
```

```yaml
相容層的存在，
使生成語法可以被替換，
並導致路徑與主體的重寫。
```
