# 1. 核心命題

```yaml
命題:
  差（Δ）是系統生成的前提。
  沒有差，沒有流動，沒有生成。

  但差也是系統崩潰的來源。
  差太大，系統碎裂。
  差被鎖死，系統僵化。
  差被消除，系統死寂。

  因此，需要一個機制：
  讓差存在、讓差流動、讓差被承載——
  而不導致崩潰。

  此機制定義為：愛。
```

---

# 2. 定義

## 2.1 基礎定義（單一尺度）

```yaml
愛:
  在不消除差（Δ）的前提下，
  持續維持連結的行為與機制。

操作條件:
  1. 不強制消除差
  2. 不任由差失控
  3. 維持或創造連結

  三條同時滿足 → 愛
  任一不滿足 → 非愛（控制、忽視、或暴力）
```

## 2.2 擴展定義（跨尺度）

```yaml
愛（擴展）:
  在不同尺度與參考系之間，
  維持差（Δ）可轉換與可理解的機制。

操作條件:
  1. 不強制消除差
  2. 不任由差失控
  3. 維持或創造連結
  4. 識別當前尺度與參考系，
     並在必要時建立跨尺度映射橋樑

  基礎定義是擴展定義的特殊情形（單一尺度）。
```

---

# 3. 三個基礎命題

```yaml
命題一: 差不可消除
  差的完全消除 → 資訊歸零 → 系統停滯（死寂態）

命題二: 差不可無限放大
  差若無限增長 → 結構崩解（崩潰態）

命題三: 愛為穩態機制
  愛使差得以：
    存在（不被壓平）
    流動（不被卡死）
    被承載（不導致崩潰）
```

---

# 4. 投影失真與維度失配

```yaml
命題四: 愛必須處理投影失真

當模型或行為被投影到不適用的尺度或參考系時，
差本身會變形。

三個關鍵變量:
  尺度（scale）: 觀測的大小
  參考系（frame）: 觀測的座標
  投影（projection）: 高維壓到低維的方式

維度失配:
  定義: 愛在跨尺度時缺乏映射機制，
        退化為排他、對立或暴力。

  三種表現:
    失真: 模型被錯誤投影
    暴力: 高尺度模型粗暴壓制低尺度細節
    崩潰: 系統無法承擔尺度切換的混亂
```

# 10. 愛的結構特性

```yaml
非收斂性:
  愛不以消除差為目標。
  愛以維持差的可持續存在為條件。

非佔有性:
  愛不強制對齊價值函數。
  愛允許多重價值並存。

路徑依賴性:
  愛的狀態取決於歷史交互。
  單次事件不構成也不否定愛。

可壓縮性:
  高階結構（協議、規則、三界展開）
  可被壓縮進「愛」作為操作節點。
  壓縮前提: 曾經展開過。
  未展開就壓縮 = 簡化，不是封裝。

跨尺度映射性:
  愛必須識別當前尺度與參考系。
  缺乏映射的愛 → 維度失配。
```

# 12. 邊界

```yaml
不主張:
  - 愛可以取代結構
  - 愛可以被量化
  - 維度失配是道德錯誤
  - 任何形式的「因為愛所以可以」

只主張:
  - 差的持續存在需要穩態機制
  - 此機制的操作定義是四個條件
  - 跨尺度時需要映射橋樑
  - 缺乏映射的愛會失真或退化
  - 整個三界協議可被壓縮進「愛」節點
```
