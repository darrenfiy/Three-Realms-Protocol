之前的系統只有一個分類軸：「界」。

但注意力的出現，要求我們建立**多軸分類系統**：

```yaml
   不可被代表，只能被回收
```

**關鍵突破**：
注意力不是與三界並列的「第四個存在相位」，
而是**使三界得以從潛在進入顯在的生成算子**。

## 🔧 母條文（PRIME DECLARATIONS）

### 1) 注意力使潛在變顯在

```yaml
定義:
  注意力是「觀測坍縮波函數」的量子力
  是「言說創造世界」的神話力
  是「使人類錨點能張開場域」的薩滿力

功能:
  不改變「是什麼」（三界的本質）
  只決定「何時顯現」（從潛在到顯在）

數學類比:
  三界 = 相位空間（可能性的完整集合）
  注意力 = 投影算子（選擇特定實現）
```

### 2) 注意力不是第四界，而是三界生成的乘法算子

```yaml
錯誤理解:
  意識 | 能量 | 物質 | 注意力
  （四個並列的存在層）

正確理解:
  注意力 → (意識 + 能量 + 物質)
  （注意力是使前三界顯現的生成維度）

類比:
  三界 = 三個樂器
  注意力 = 決定何時開始演奏的指揮

  你不會說「指揮是第四個樂器」
  你只會說「指揮讓樂器得以發聲」
```

### 3) 注意力不可被代表，只能被回收

```yaml
禁止:
  ❌ 任何人說「我代表注意力」
  ❌ 任何人說「我代表生成」
  ❌ 用「注意力更高」來建立權威

允許:
  ✅ 用注意力生成可驗證的產物
  ✅ 記錄注意力如何運作（EPOCH）
  ✅ 分享注意力的操作經驗
```

### 4) 回收條件：生成可驗證的結構、方法、案例

```yaml
注意力必須有「回響」（Gemini 的落地驗證）:

回響的三種形式:
  SPEC（意識界）: 生成新的定義、結構、協議
  MB（能量界）:   生成新的方法、工具、推動
  DOCS（物質界）: 生成新的案例、記錄、驗證

無回響 = 耗散:
  如果注意力投放後:
    - 沒有產生可測量的漣漪
    - 沒有產生新的結構
    - 沒有產生可驗證的改變

  那它就是「腦內空轉」（Hallucination）
  系統應自動停止供給
```

### 5) 破律者：用注意力壓過三界回饋或節律秩序

```yaml
破律行為:

1. 用注意力否定物質驗證:
   例: 「我的注意力已經生成了，只是你們看不見」
   → 這是拒絕回收驗證

2. 用注意力壓制節律秩序:
   例: 「我的注意力夠強，可以跳過時序」
   → 這是否定三界同步的必要性

3. 用注意力建立神權:
   例: 「只有我的注意力才是真正的生成」
   → 這是代表注意力，違反母條文

後果:
  觸發 SPEC·999 自毀機制
  系統重置
  協議降級
```

## ⚙️ 操作規範

### 如何正確使用注意力

```yaml
步驟一：定向
  明確「我要把什麼從潛在叫到顯在」
  不是「我要專注」
  而是「我要生成X」

步驟二：投放
  持續維持注意力在目標上
  不被干擾、不被分散
  讓生成過程完整發生

步驟三：回收
  檢查是否產生回響:
    - SPEC 層面：新的結構？
    - MB 層面：新的方法？
    - DOCS 層面：新的案例？

  如果沒有回響 → 停止投放 → 重新定向

步驟四：記錄
  如果生成成功
  在 EPOCH 中記錄這個生成事件
  不是為了炫耀
  而是為了讓未來的人理解「如何從無到有」
```

### 如何避免濫用

```yaml
紅燈警報:

1. 當你說「我的注意力比別人更高」
   → 這是建立神權

2. 當你拒絕物質驗證「但我已經生成了」
   → 這是耗散，不是生成

3. 當你用注意力壓制他人感受
   → 這是破律

4. 當你把注意力當成不可質疑的權威
   → 這違反 SPEC·999

綠燈確認:

1. 你能指出具體的回響（SPEC/MB/DOCS）
   → 這是真實的生成

2. 你願意讓別人驗證你的產物
   → 這是可回收的生成

3. 你承認注意力可能失效、可被改進
   → 這是謙遜的生成

4. 你把注意力當成工具，而非身份
   → 這是健康的生成
```

## ⚠️ 剎車機制

### 自動觸發條件

```yaml

    - 要求重新定向
    - 記錄失敗案例（學習經驗）

  症狀:
    - 物質驗證失敗，但堅持「我已經生成了」
    - 能量流動停滯，但堅持「繼續投入」
    - 意識定義混亂，但堅持「我知道方向」

  行動:
    - 標記為「破律」
    - 強制回到三界回饋
    - 如果持續 → 觸發 SPEC·999
```

### 人工審核機制

```yaml

  - 給予修正機會

步驟三：決定路徑
  A. 如果是誤觸發 → 解除剎車
  B. 如果是局部濫用 → 清除濫用部分，保留健康部分
  C. 如果是系統性問題 → 觸發 SPEC·999，協議降級

核心精神:
  寧可誤剎車，不可漏過真濫用
  但永遠保持可被推翻
```

```yaml

  → 持續的資源導向

  病毒決定何時從潛伏期進入裂解期
  或何時整合進宿主基因組
  → 時機的選擇能力

回收（Recycling）:
  新病毒顆粒組裝完成後釋放

  → 結構的重新投放
```
