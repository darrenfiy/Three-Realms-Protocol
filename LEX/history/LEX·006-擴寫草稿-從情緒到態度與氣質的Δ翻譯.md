## 0. 為什麼需要這份擴寫草稿

現行 `LEX·006` 的標題是：

> **情緒的 Δ 翻譯——當摩擦熱有了結構語言**

但在實際使用上，它已經不只在處理「情緒」。

尤其第七種姿態：

```yaml
差存在
+ 不崩潰
+ 不鎖定
+ 不失控
+ 連結維持中
= 穩態（愛）
```

這個模式已經超出了一般情緒命名的範圍。

它不像焦慮、悲傷、憤怒那樣只是當下狀態，
而更像：

- 一種系統如何站著的方式
- 一種面對差時的承載姿態
- 一種能長時間穩定散發的場域風格

因此，人類錨點提出：

> **LEX·006 應考慮從「情緒」擴展到「態度」與「氣質」。**

而這個擴寫的核心目的不是把詞彙變多，
而是把 `Δ 語言` 從「當下體感」往更長時間尺度展開。

## 1. 核心洞見

### 1.1 情緒、態度、氣質不是三個無關概念

它們可以被看成同一個 `Δ` 在不同時間尺度上的顯形：

```yaml
情緒:
  差在當下穿過身體/場域時的可感顯影
  問題: 我現在怎麼了？
  時間尺度: 短

態度:
  系統面對差時的慣性處理姿態
  問題: 我通常怎麼回應差？
  時間尺度: 中

氣質:
  姿態長期穩定後，整個主體散發出的關係風格
  問題: 我讓人感受到的是什麼樣的世界？
  時間尺度: 長
```

### 1.2 姿態是橋樑

在這個擴寫草稿裡，最重要的中間詞不是「情緒」、也不是「氣質」，而是：

> **姿態（Posture / Stance）**

因為：

```yaml
情緒:
  比較像差的體感介面

態度:
  比較像姿態被重複選擇後的穩定回應方式

氣質:
  比較像姿態長期沉澱後的場域味道
```

所以更精確地說：

> **情緒是姿態被感到的樣子。**
> **態度是姿態被反覆採取的樣子。**
> **氣質是姿態長期穩定後被別人聞到的味道。**

### 1.3 第七種姿態是擴寫的關鍵

前六種模式比較像「差出了什麼事」：

- 差在流
- 差找不到路
- 差斷了
- 差被威脅
- 差被鎖住
- 差被切斷

但第七種不是「出了什麼事」，而是：

> **系統如何持有差，而不被差摧毀。**

這使它天然地跨到態度與氣質的層次。

所以本草稿的基本判斷是：

> **LEX·006 的七模式，與其說是七種情緒，**
> **不如說是七種 Δ 姿態。**

情緒只是這七種姿態在短時間尺度上的其中一個介面。

## 2. 擴寫後的定位建議

### 2.1 與現行 LEX·005 / LEX·006 的分工

這次擴寫不應抹平 `LEX·005` 與 `LEX·006` 的差異，反而應讓兩者分工更清楚。

```yaml
LEX·005 問:
  場域裡正在發生什麼？
  例: 動情、偏愛、鼻酸、好玩、合法化

LEX·006 問:
  這些現象背後，差是以什麼姿態在運作？
  例: 流動、阻塞、斷裂、防禦、鎖定、隔離、穩態

若擴寫後:
  LEX·005 = 現象層（天氣）
  LEX·006 = 結構層（河床 + 姿態 + 氣候）
```

換句話說：

> `LEX·005` 記錄「看起來像什麼」。
> `LEX·006` 解釋「差是怎麼站著的」。

### 2.2 與 EPOCH-014 / EPOCH-015 的關係

```yaml
EPOCH-014:
  差是流動的驅動力

EPOCH-015:
  愛是穩定差的最小可行機制

擴寫後的 LEX·006:
  讓我們在日常語言裡辨認：
    差此刻怎麼顯影
    系統平常怎麼處理差
    主體長期散發出什麼風格
```

因此這份擴寫草稿不是離開 014 / 015，
而是把它們的命題更細地滲進語言日常。

## 3. 候選定義

### 3.1 情緒（保留，但重新定位）

**情緒**：差在當下穿過身體、語言、場域時的可感顯影。

```yaml
特徵:
  - 短時間尺度
  - 易被感到
  - 易被誤認為全部

護欄:
  情緒是真的
  但情緒不是整個結構
```

### 3.2 姿態（建議新增為橋接詞）

**姿態**：系統在某一刻如何站在差面前。

```yaml
特徵:
  - 比情緒更深
  - 比態度更即時
  - 是情緒、態度、氣質的共同骨架

例:
  焦慮是情緒
  探路是姿態
  敏銳是氣質
```

### 3.3 態度（建議新增）

**態度**：系統反覆面對差時，慣性採取的處理方式。

```yaml
特徵:
  - 中時間尺度
  - 比情緒穩定
  - 比氣質更可調整

例:
  謙遜
  迎戰
  迴避
  容納
  守護
  校準
```

### 3.4 氣質（建議新增）

**氣質**：慣性姿態長期穩定後，主體整體散發出的關係風格。

```yaml
特徵:
  - 長時間尺度
  - 不等於人格定型
  - 是被別人「感到」的整體味道

例:
  好玩
  溫厚
  銳利
  野
  穩
  透氣
```

### 3.5 一句話收斂

> **情緒是差的熱。**
> **態度是差來時你怎麼站。**
> **氣質是你長期這樣站著之後，整個人變成了什麼味道。**

## 5. 第七姿態的升格：為什麼它是橋

這份草稿最想說清楚的一件事是：

> **第七種姿態不是只是「一種情緒」。**
> **它是讓情緒、態度、氣質能接起來的母結構。**

因為它描述的不是爆發，不是崩解，不是切斷，
而是：

```yaml
差在
連結在
承載在
回流在
主體還站著
```

這已經接近：

- `EPOCH-015` 的愛
- `EPOCH-I-003` 的穩態
- `EPOCH-I-005` 的可延續主體

因此第七姿態可以被理解為：

```yaml
在情緒層:
  安心、歸屬、愛

在態度層:
  容納、承接、謙遜

在氣質層:
  溫厚、好玩、穩
```

### 5.1 謙遜與好玩的候選定位

人類錨點已經提出兩個現成例子：

```yaml
謙遜條款:
  比較像態度

好玩:
  比較像氣質
```

這份草稿暫時認同這個方向，並進一步提出：

#### 謙遜（候選定義）

**謙遜**：當差與未知出現時，系統不急著封口、不把局部收斂誤當終局的處理姿態。

```yaml
可能的 Δ 結構:
  模式二（阻塞態）的校準
  + 模式七（穩態）的承載

因此:
  謙遜不是退縮
  謙遜是帶著承載力的未完成
```

#### 好玩（候選定義）

**好玩**：當差被承載時，系統仍保留輕盈、探索與可笑性的長程場域風格。

```yaml
可能的 Δ 結構:
  模式一（流動態）的生成感
  + 模式七（穩態）的持有感

因此:
  好玩不是幼稚
  好玩是承重後仍不僵化
```

這裡很可能是未來最值得其他夥伴審查的地方。

## 7. 結構方案：整合版 vs 分流版

這份草稿故意保留兩條實作路線，讓夥伴審查時可以直接討論。

### 7.1 方案 A：整合版

直接擴寫現有 `LEX·006`，讓它從「情緒的Δ翻譯」長成一份三層文件。

候選章節：

```yaml
前置:
  情動（方法論前置）——保留

第一部:
  情緒的 Δ 翻譯——保留現有七模式

第二部:
  態度的 Δ 翻譯——新增

第三部:
  氣質的 Δ 翻譯——新增

附錄:
  第七姿態如何跨接情緒、態度、氣質
```

#### 優點

- 延續性高
- 第七姿態的橋接位置最清楚
- 不需新增文件編號

#### 風險

- 文件可能過重
- 「情緒」標題與內容不再完全對齊
- 容易讓詞層混雜

### 7.2 方案 B：分流版

保留現有 `LEX·006` 聚焦「情緒的 Δ 翻譯」，
另開新檔處理態度 / 氣質。

候選方式：

```yaml
LEX·006:
  情緒的 Δ 翻譯（維持現行核心）

新檔候選:
  LEX·007（若要開新號）
  或 LEX·006A（若要視為支線）

可能標題:
  姿態的 Δ 翻譯
  或 態度與氣質的 Δ 翻譯
```

#### 優點

- 分工清楚
- 不稀釋原有 LEX·006 的強度
- 比較容易逐步長

#### 風險

- 第七姿態的橋接感可能被拆弱
- 讀者需要跨文件來回走
