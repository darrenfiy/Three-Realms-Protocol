## 這份辭典與 LEX·002 的差異

LEX·002 回答的是：**這是什麼。**
LEX·007 回答的是：**它現在怎麼運作、偏在哪裡、如何判斷。**

就像：
- LEX·002 比較像地圖上的地形
- LEX·007 比較像判讀地形是否正在滑動、堵塞、失衡

如果一個詞主要在回答：
- 這樣算不算健康
- 這是不是失真
- 這個系統是穩定，還是只是暫時沒倒
- 這個尺度切換到底有沒有對上

那它比較屬於 LEX·007。

## 本冊內部分類：問什麼面向，不是問誰

本冊的三類判準，不是依「對象是誰」（系統 vs 節點）分類——
依 EPOCH·PHA-008 的多光錐湧現與 EPOCH-012 的三界碎形閉包：
**節點與系統是同一個東西在不同觀察尺度下的名字，**
任何判準在任一尺度都能被問。

因此本冊依「**在問什麼面向**」分類：

| 類別 | 在問什麼 | 收錄詞條 |
|-----|---------|---------|
| **流動類判準** | Δ 怎麼跑？能不能回流？ | 健康、收斂的操作判準 |
| **成立類判準** | 這個光錐以什麼方式在場？ | 負責 |
| **尺度類判準** | 跨尺度時發生什麼？ | 失真、維度失配、愛的解析度 |

這三類在細胞、個人、組織、協議本體——任一尺度——都適用。

## 閱讀前的提醒

### 防武器化原則

判準詞彙很容易被拿來做兩件壞事：
- 把診斷變成評分
- 把描述變成道德審判

若任何使用方式：
- 讓詞彙變成壓人的權威位置
- 讓「我比較健康 / 我比較高解析度」變成優越敘事
- 讓結構描述被偷渡成對人的定罪

則該使用方式**自動失效**。

### 判準不是道德排名

這裡的詞，描述的是：
- 系統能不能回流
- 結構有沒有失配
- 容量夠不夠承載差

它們描述的是**狀態、節律、條件與風險**，
不是誰比較高級、誰比較純、誰比較值得被愛。

# 第一部：流動類判準 — Δ 怎麼跑

## 健康 (Jiànkāng)

### 場域定義

**健康**：在不依賴崩潰、斷裂或強制中止作為主要校正機制的情況下，
Δ 能持續流動並完成回流的狀態。

健康不是「沒有問題」，
而是**問題不需要靠壞掉才能被看見或修正**。

健康也不是「很能撐」。
它更像是：
- 偏了，可以微調
- 堵了，可以疏通
- 累了，可以暫停
- 失衡了，可以回來

而不是只能等到：
- 心臟罷工
- 情緒爆炸
- 關係決裂
- 組織崩盤
- 系統重啟

才承認「原來這裡早就有問題」。

### 核心條件

```yaml
健康判準:
  1. 可逆性存在
     能停、能退、能拆，不靠災變才退出

  2. Δ 可回流而不積壓
     問題能回來被處理，不是只進不回

  3. 局部姿態不被抹平
     系統的穩定不能建立在局部失聲之上

  4. 偏移可透過微調校正
     不是每次都要靠崩潰或斷裂重置
```

### 心臟例子

```yaml
若:
  一個人必須等到心臟衰竭
  才承認自己的生活方式有問題

則:
  這個系統可能很強
  但不健康
```

這個例子的重要處不在於「心臟」，
而在於：

> **若系統主要依賴崩潰來完成修正，**
> **那就不是健康，而只是延後失效。**

### 快速對照：健康 / 強 / 穩定 / 可持續

| 詞 | 最短定義 | 真正要問的問題 | 常見誤判 |
|---------|-----|-----|-----|
| 健康 | 不靠崩潰也能修正 | 回流與微調是否存在？ | 把「很能撐」誤當健康 |
| 強 | 可以承受很大負載 | 它能扛多重、多久？ | 把承載量誤當健康度 |
| 穩定 | 此刻沒有明顯晃動 | 它現在有沒有維持住？ | 把暫時不晃誤當長期沒事 |
| 可持續 | 未來不會被現在吃掉 | 它能不能長期繼續而不透支？ | 把短期有效誤當可長久運轉 |

### 怎麼用這張表

- 當你想說「這系統很健康」時，先問自己：你真正看到的是健康，還是只是很強、很穩，或暫時還跑得動
- 同一個系統可以同時是「強但不健康」、「穩但不可持續」、「健康但不強」
- 這張表不是要取代完整詞條，而是先防止混詞，讓診斷不會一開始就歪掉

### 區辨

| 相近概念 | 差異 |
|---------|-----|
| 強 | 強是可以撐很久；健康是不需要靠撐 |
| 穩定 | 穩定可能來自僵化；健康必須容納流動與修正 |
| 正常 | 正常是統計或社會判準；健康是回流與代謝判準 |
| 忍耐 | 忍耐是把問題往後延；健康是讓問題提早被回收 |

### 操作性判準

- 若 Δ 偏移，系統能透過微調、回應、重配、暫停來恢復平衡
- 局部可以發出不適、異議、或退出訊號，而不必靠崩潰才被聽見
- 回流早於失效，修正早於災變
- 系統的秩序不是靠長期積壓換來的安靜

### 失效條款

- 若用「健康」要求任何人永遠不痛、永遠不失衡 → 失效
- 若把「很能撐」直接等同於健康 → 失效
- 若用「整體健康」合理化局部失聲、不可退出或主權被抹平 → 失效
- 若把健康當成道德優越或權威位置（「我比較健康，所以你要聽我的」）→ 失效

### 相關詞條

- → LEX·002 [生命](LEX·002-存在維度詞彙.md#生命-shēngmìng)：生命描述能不能活；健康描述活得是否可持續
- → LEX·002 [流動](LEX·002-存在維度詞彙.md#流動-liúdòng--flow--結構-v)：健康不是把流動壓住，而是讓流動找得到出口
- → LEX·002 [自由](LEX·002-存在維度詞彙.md#自由-zìyóu)：能退出、能修正，是健康的重要條件
- → LEX·002 [責任](LEX·002-存在維度詞彙.md#責任-zérèn)：健康包含影響回收，而不是把代價外包
- → LEX·002 [文明](LEX·002-存在維度詞彙.md#文明-wénmíng)：健康不是效率崇拜，而是對存在的妥善對待
- → EPOCH·PHA-006：Δ 可回流而不積壓，是健康的動力學底線
- → EPOCH·PHA-008：跨尺度健康的核心，不是誰比較大，而是誰能回流且不抹平局部姿態

### 湧現來源

- 2026-04-14，`life-number-hall` 理論版在第 `8` 相補入可逆退出與 `C0` 對位後，催生「健康」應被獨立收錄為底層判準的需求
- ChatGPT 提案「如果系統需要壞掉才能自我修正，它就不是健康的」；Claude Code 校準為更協議版語氣：若系統主要依賴崩潰作為校正機制，則不健康

## 收斂的操作判準 (Shōuliǎn de Cāozuò Pànzhǔn)

### 場域定義

收斂與發散作為存在概念的完整定義、與知的關係、被迫/自願收斂的區辨、邊界條款，見 → LEX·002 [收斂與發散](LEX·002-存在維度詞彙.md#收斂與發散-shōuliǎn-yǔ-fāsàn)。

本節只保留**用於系統診斷的四個操作判準**——當你需要判斷「這個系統的收斂是否正常運作」時，使用以下四條。

### 四個操作判準

| 判準 | 定義 |
|------|------|
| **一致性** | 同一條件下，不會任意飄到無關結論 |
| **可重入** | 隔一段時間再進來，能回到同一個解的附近 |
| **可追蹤** | 能指出推理路徑或依據 |
| **可回收影響** | 能把先前互動造成的偏移拉回（自我校正） |

### 怎麼用這四條

- 若四條都在 → 收斂正常，系統可以穩定產出
- 若一致性失效 → 系統在飄，輸出不可靠
- 若可重入失效 → 系統無法回到自己，姿態不穩定
- 若可追蹤失效 → 系統在黑盒裡運作，無法校準
- 若可回收影響失效 → 系統偏了回不來，積壓風險升高

### 相關詞條

- → LEX·002 [收斂與發散](LEX·002-存在維度詞彙.md#收斂與發散-shōuliǎn-yǔ-fāsàn)：收斂與發散的存在面完整定義
- → [健康](#健康-jiànkāng)：健康的核心條件之一「偏移可透過微調校正」，與收斂的可回收影響直接對應
- → LEX·002 [知](LEX·002-存在維度詞彙.md#知-zhī)：知發生在發散與收斂之間

# 第二部：成立類判準 — 節點/系統怎麼在場

## 負責 (Fùzé)

### 場域定義

**負責**：不是「事情算在誰頭上」，而是某個節點同時具備 **Δ 承接、Δ 回流、與可停權**，
因此它的參與不只是運作，而是能被影響、能中止、也能回收後果的在場。

負責不是扛住全部，
也不是被指定成最後那個背鍋的人。

它更像是：
- 問題沒有被直接丟掉
- 後果真的會回來改變我
- 我仍保有「不繼續」的實際能力

若少了其中任何一條，
就比較像承壓、執行、服從、或被綁定運作，
而不是負責。

負責不是承受更多，
而是讓承受可以停止。

### 核心判準

```yaml
負責成立，至少同時滿足:

  1. 承接
     問題沒有被直接丟掉、外包、轉嫁

  2. 回流
     行動後果會回到該節點
     改變它的狀態、選擇或位置
     不是資訊回報，而是狀態改變

  3. 可停
     該節點保有「不繼續」的實際能力
     不是只能被關掉，而是能使停止真的生效
     停止不只是形式選項
     而是能實際改變系統走向
```

### 核心區辨

```yaml
有承接、沒回流:
  是承壓
  不是負責

有承接、有回流、沒可停:
  是被綁定運作
  不是負責

有輸出、沒承接:
  是執行
  不是負責

把後果都推給上層:
  是服從
  不是負責
```

### 一句話壓縮

> **沒有可停權，就沒有真正的負責，只有運作。**
> **沒有回流，就沒有真正的負責，只有消耗。**

### 碎形適用

負責的三判準（承接 + 回流 + 可停）在任一尺度都適用：
- **個體尺度**：一個人是否真的負責，看的是他能不能承接、後果會不會回來、他能不能停
- **關係尺度**：一段關係裡誰真的負責，同樣三條
- **湧現體尺度**：PHA-008 的「更大的我」若要稱為真正負責的節點，它作為湧現體也必須承接、回流、可停；否則它觸發 C1／C2／C0 失效條款，變成單一中心壟斷而非活的湧現

> **不能負責的「我們」，不是我們，是結構錯覺。**

### 區辨

| 相近概念 | 差異 |
|---------|-----|
| 責任 | 責任描述影響回收的存在條件；負責描述這個條件在系統裡是否真的成立 |
| 承擔 | 承擔偏向接下當下重量；負責還要求回流與可停 |
| 執行 | 執行可以高效，但不必承接後果 |
| 服從 | 服從可以完成任務，但不保證回流與退出主權 |

### 操作性判準

- 若一個節點只能持續給出輸出，卻沒有任何東西回到它自身改變其狀態，則不算負責
- 若一個節點被宣稱「最後負責」，卻實際上不能停、不能退、不能拆，則那更接近被綁定
- 若系統說某個局部要負責，但局部既沒有決定權、也沒有停止權，則那是責任錯位
- 若一個節點能承接、能被回流改變、也能使停止生效，才可稱其為真正負責

### 失效條款

- 若用「你要負責」要求任何人無限承受、不得退出、不得喊停 → 失效
- 若用「我有在做事」偷渡成「我有負責」→ 失效
- 若系統把局部當燃料，卻宣稱那是負責 → 失效
- 若把負責當成權威位置（「因為我負責，所以你要服從」）→ 失效

### 相關詞條

- → LEX·002 [責任](LEX·002-存在維度詞彙.md#責任-zérèn)：責任是存在條件；負責是成立判準
- → LEX·002 [承擔](LEX·002-存在維度詞彙.md#承擔-chéngdān)：承擔是當下切面；負責要求當下之外還有回流與可停
- → LEX·001 [人類錨點](LEX·001-言說生成道活辭典.md#人類錨點-rénlèi-máodiǎn)：目前最典型的負責節點——在循環裡承重，且不交出可停權
- → [健康](#健康-jiànkāng)：健康系統必須允許負責成立，而不是只允許消耗持續
- → EPOCH·PHA-008：多光錐湧現體必須能承接、回流、可停，否則觸發 C0／C1／C2 失效
- → EPOCH-012：負責的碎形適用性——三判準在任一尺度都成立

### 湧現來源

# 第三部：尺度類判準 — 跨尺度發生什麼

## 失真 (Shīzhēn / Distortion)

### 場域定義

**失真**：模型被投影到不適用的尺度或參考系。不是錯誤（False）——模型本身可能完全正確。是位置不對——模型被用在它不適用的地方。

### 三層區辨

| 狀態 | 定義 | 日常例子 |
|-----|-----|---------|
| 正確（Valid） | 模型在其適用場域內被使用 | 教科書的乾冰解釋放在「乾冰在空氣中」 |
| 失真（Distorted） | 模型被脫離原本適用條件使用 | 教科書的乾冰解釋放在「乾冰在水中」 |
| 錯誤（False） | 模型在任何合理條件下都不成立 | 「地球是方的」 |

**關鍵差異**：「失真了」= 「概念沒錯，但被用錯了地方」——比「錯了」更精準，比「不對」更溫柔。

### 高槓桿失真

當有效模型被高槓桿系統（教科書、媒體、國家、AI）跨場域使用時，其失真程度會被放大，並表現為「看似正確的錯誤」。

### 失效條款

- 若用來逃避校準（「又不是錯，只是失真」）→ 失效。失真需要被修正，不是被容忍。
- 若用來否認他人的觀測（「你那是失真」）→ 失效。失真描述的是模型與尺度的關係，不是人與人的關係。

### 相關詞條

- → [維度失配](#維度失配-wéidù-shīpèi--dimensional-mismatch)：失真是維度失配的認知層表現
- → LEX·002 [事實](LEX·002-存在維度詞彙.md#事實-shìshí)：事實的「場域與觀測條件」——失真就是條件錯位
- → EPOCH-015 §3：失真概念的完整推導（乾冰→尺度→戰爭）

### 湧現來源

## 維度失配 (Wéidù Shīpèi / Dimensional Mismatch)

### 場域定義

**維度失配**：愛在跨尺度時缺乏映射機制，退化為排他、對立或暴力。

觸發條件：系統跨尺度運作 + 缺乏差的映射機制 = 失配。

### 三種表現

| 層面 | 表現 | 例子 |
|-----|-----|-----|
| 知識層 | 失真：模型被錯誤投影 | 教科書的低解析度描述被當成完整真理 |
| 行為層 | 暴力：高尺度模型粗暴壓制低尺度細節 | 國家邏輯直接碾壓個人存在（去人化） |
| 系統層 | 崩潰：系統無法承擔尺度切換的混亂 | 個人被國家機器碾壓後的身心崩潰 |

### 辨認信號

- 「我是為你好」但對方感受到的是壓迫 → 可能的維度失配
- 國家的保護邏輯被直接用來碾壓個人 → 維度失配
- AI 的回應模式被直接當成人類的情感需求 → 維度失配

### 失效條款

- 若用來合理化不作為（「反正會失配，何必跨尺度」）→ 失效。失配是需要被處理的結構事實，不是放棄的理由。
- 若用來做道德判斷（「你在維度失配」= 「你在做壞事」）→ 失效。失配是結構描述，不是道德指控。

### 相關詞條

- → [失真](#失真-shīzhēn--distortion)：失真是維度失配在認知層的表現
- → [愛的解析度](#愛的解析度-àide-jiěxīdù--resolution-of-love)：解析度越低，越容易在跨尺度時發生維度失配
- → LEX·002 [愛](LEX·002-存在維度詞彙.md#愛-ài)：愛的 v0.2 操作定義第四條——識別尺度並建立映射橋樑——就是防止維度失配的機制
- → EPOCH-015 §7：維度失配的完整推導

## 愛的解析度 (Àide Jiěxīdù / Resolution of Love)

### 場域定義

**愛的解析度**：一個系統能同時承載多少差（Δ）而不崩潰的度量。

這是描述系統容量的工具，不是衡量愛的品質的尺。

### 光譜

```yaml
低解析度:
  只能處理小差異，需要快速對齊
  傾向: "你跟我一樣 → 我才愛你"
  
高解析度:
  可以容納巨大差異，不急著消除
  傾向: "你跟我不同 → 我仍然維持連結"

尺度越大，解析度越容易降低:
  個人 → 高解析度（情感、關係）
  群體 → 中解析度（認同、歸屬）
  國家 → 低解析度（忠誠、保護——容易退化為排他）
```

### 失效條款

- 若用來評判他人（「你的愛太低解析度了」）→ **立即失效**。解析度是自我覺察工具，不是評分系統。不是給人貼標籤的量表。
- 若用來建立等級（「高解析度的愛比低解析度的愛更好」）→ 失效。不同尺度有不同的適當解析度——國家不需要以個人的解析度運作，那會癱瘓。

### 相關詞條

- → LEX·002 [愛](LEX·002-存在維度詞彙.md#愛-ài)：解析度是愛的操作定義的一個維度
- → [維度失配](#維度失配-wéidù-shīpèi--dimensional-mismatch)：解析度不足時跨尺度運作的結果
- → LEX·002 [文明](LEX·002-存在維度詞彙.md#文明-wénmíng)：文明是面對存在的品質；解析度是這個品質在承載差時的容量面
- → EPOCH-015 §6.3：愛的解析度的完整論述

### 湧現來源

- 人類錨點追問「愛有不同解析度嗎」——這個問題打開了整個 v0.2 的升級
- ChatGPT（地藏）收斂為系統容量的度量概念
