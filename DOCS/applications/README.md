## 🎯 應用開發原則

### 1. 遊樂園精神

```yaml
✅ 應該是:
  - 好玩的（不是功課）
  - 可選的（不是義務）
  - 有邊界的（可退出、可逆）
  - 謙遜的（SPEC·999，可能錯）

❌ 不應該是:
  - 強迫的
  - 競爭的
  - 排名的
  - 神化的
```

### 2. 兩層設計

每個應用應該考慮：
- **外層**：公開、輕鬆、低門檻（給所有訪客）
- **內層**：深度、專屬、高密度（給協議身體成員）

### 3. 資訊密度分層

- **遊樂園**（PLAYGROUND.md）：超低密度，邀請式
- **應用介面**：低密度，易用性優先
- **協議文檔**（SPEC/EPOCH）：高密度，完整性優先

## 🚀 如何新增應用

### 1. 創建資料夾

```bash
mkdir DOCS/applications/你的應用名稱
```

### 2. 基本檔案結構

```
你的應用名稱/
├── README.md       （總覽與快速開始）
├── theory.md       （理論基礎，可選）
└── ...             （其他檔案）
```

### 3. 更新此 README

在「當前應用」區塊加入你的應用。

### 4. 連結到遊樂園

如果是公開設施，更新 `PLAYGROUND.md` 加入入口。
