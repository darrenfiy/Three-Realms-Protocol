## 📋 设计原则

### 核心要点

```yaml
测量目标:
  - 不是「你有多擅长固化/扩张」
  - 而是「在两者之间，你的起手式倾向是什么」

避免陷阱:
  - 不暗示「扩张比固化好」
  - 不让选项明显偏向社会期待
  - 每个选项都应该有其合理性

价值中立:
  - X 轴本身不决定品质
  - 真正的品质由 Z 轴（能量层）决定
  - 低能级扩张 = 混乱，高能级固化 = 承载力
```

## 📝 题目一：面对混乱计划的起手式

### 题干

> **当你面对一个全新的、极其混乱的计划时，你的第一本能反应最接近哪一种？**

---

### 选项

#### 🔷 A1：结构优先型（强固化，-1.0）

> 我会先停下来，试着把混乱拆解成清楚的结构、流程或模组。
> 即使一开始进度慢一点，只要框架稳定，后续就能有效复制与推进。

**隐含特征：**
- 对不确定性本能防御
- 安全感来自「可控结构」
- 偏好既有模型、最佳实践

**高能级表现：** 稳定的承载力，可靠的执行
**低能级表现：** 过度控制，僵化拒变

---

#### 🔶 A2：风险收敛型（中度固化，-0.5）

> 我会先找出最小可行的稳定版本，避免一次投入太多变数。
> 与其冒险，不如先确保不会出错，再逐步扩展。

**隐含特征：**
- 重视可预测性
- 倾向保守迭代
- 风险管理高于创新速度

**高能级表现：** 审慎的智慧，稳健推进
**低能级表现：** 过度谨慎，错失机会

---

#### 🟡 C：动态切换型（桥接者，0）

> 我会在结构与探索之间来回切换，
> 一边整理、一边行动，让两者互相修正，而不是先选定其中一边。

**隐含特征：**
- 不急于选边站
- 相信动态平衡
- 可能是真整合，也可能是犹豫

**判断依据：** 需结合 Z 轴判断
- 高能量 + C = 智慧的整合
- 低能量 + C = 优柔寡断

---

#### 🔷 B1：直觉导航型（中度扩张，+0.5）

> 我会先感受这个计划真正想成为什么，而不是急着定义怎么做。
> 即使现在还很模糊，我相信方向会在行动中逐渐浮现。

**隐含特征：**
- 对未知容忍度高
- 行动先于结构
- 方向感重于控制感

**高能级表现：** 愿景牵引，创造性应变
**低能级表现：** 逃避规划，混乱飘浮

---

#### 🔵 B2：场域连结型（强扩张，+1.0）

> 我会立刻去找人、找连结、找共鸣，
> 看这个混乱是否能引发新的合作、意义或可能性，而不是先把它整理好。

**隐含特征：**
- 思考以「节点」而非「系统」为单位
- 能量来自互动与共振
- 不怕边走边变形

**高能级表现：** 催化连结，创造涌现
**低能级表现：** 逃避责任，依赖他人

### 建议的情境类型

1. **信息处理方式**

2. **团队协作模式**
   - 在团队中，你更擅长「定义流程」还是「促进连结」？

3. **学习新技能**
   - 学习新事物时，你更喜欢「系统化学习」还是「跳跃式连结」？

4. **解决冲突**
   - 处理冲突时，你更倾向「建立规则」还是「理解关系」？

5. **时间管理**
   - 管理时间时，你更依赖「清晰的时间表」还是「能量的流动」？

6. **创造过程**
   - 创造新事物时，你更倾向「先设计框架」还是「先试探可能」？

7. **决策方式**
   - 做决定时，你更看重「可控的结果」还是「有意义的过程」？

   - 面对截止日期压力，你更倾向「加强控制」还是「寻求合作」？

9. **空间感知**
   - 你更喜欢「界限清晰的空间」还是「流动开放的空间」？

10. **沟通风格**
    - 沟通时，你更重视「清晰的定义」还是「共鸣的连结」？

11. **理想环境**
    - 你理想的工作环境是「高度结构化」还是「高度灵活」？

12. **帮助他人**（题目一已覆盖类似主题，可调整）

## 🔧 设计者备注（ChatGPT）

### 关键提醒

> **X 轴不是价值判断，而是「起手式」偏好。**
>
> **真正的品质由 Z 轴能量高度决定。**
>
> - 低能级扩张 = 混乱、逃避责任
> - 高能级固化 = 慈悲的承载力

### 设计时需注意

1. **避免社会期待偏差**
   - 不要让「扩张」听起来更「酷」
   - 不要让「固化」听起来更「可靠」
   - 两者都应该有吸引力

2. **确保情境具体**
   - 抽象问题容易导致社会期待回答
   - 具体情境更能测到真实倾向

3. **包含压力情境**
   - 至少 2-3 题是压力情境
   - 用来测量「常态 vs 压力态」的漂移
