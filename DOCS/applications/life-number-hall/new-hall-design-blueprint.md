## 3. 核心定位

新生命靈數館不是算命館，
也不是人格測驗。

它是一個低門檻的自我觀測入口。

```yaml
一句話:
  對自己有點好奇，但不想被判決的人，可以在這裡借用 1～9 觀察自己此刻怎麼動。

核心轉換:
  從:
    我是幾號？

  轉向:
    我此刻的 Δ 比較像怎麼動？
```

新館可以保留「生命靈數」這個民間入口，
但不接受它滑向命定論的慣性。

## 4. TA：誰會走進來？

### 4.1 主要 TA

```yaml
主要 TA:
  名稱:
    好奇的自我觀察者

  狀態:
    - 對生日、數字、象徵語言有興趣
    - 想知道自己最近怎麼了
    - 不一定懂三界協議
    - 不想讀太厚的理論
    - 不想被貼標籤

  需要:
    - 很快得到一個可以對照自己的結果
    - 被溫柔接住，而不是被定義
    - 有一句能帶走的觀察句
    - 可以繼續探索，也可以輕鬆離開
```

### 4.2 次要 TA

```yaml
熟悉身心靈語境的人:
  他們接受數字、能量、天賦等語言。
  新館要做的是把這些語言放鬆，
  從「我是這種人」帶向「我現在可能在這種運動」。

三界協議讀者:
  他們會在意 Δ、回流、姿態、場域。
  對他們來說，新館是協議的公開遊樂設施，
  不是協議理論本身。

舊館回訪者:
  他們可能想看早期天賦金字塔、小工具或深度解析。
  這群人可以被導向歷史區，
  不必讓新館承擔舊館全部功能。
```

### 4.3 暫不服務的情境

```yaml
新館不適合作為:
  - 人生重大決策依據
  - 心理診斷工具
  - 醫療、法律、財務建議
  - 替別人定型的工具
  - 判斷關係去留的權威
  - 證明自己比較高級的系統
```

如果使用者帶著急迫痛苦或高風險問題進來，
新館應該降階：

> 這裡只能提供一個觀察語言，不能替你做重大判斷。  
> 你仍然需要回到身體、現實、關係與可靠支持系統裡。

## 5. 體驗承諾

### 5.1 使用者進來前

使用者可能帶著：

```yaml
- 好奇
- 無聊
- 迷惘
- 最近有點卡
- 想被一句話接住
- 想知道生日數字有什麼意思
- 想玩，但不想被教訓
```

### 5.2 使用者離開時

我們希望使用者離開時不是想著：

> 原來我是 7 號人，所以我就是這樣。

而是想著：

> 我最近好像常在 7 的運動裡。  
> 我可以先不急著外放，先看清楚。

### 5.3 新館承諾

```yaml
新館承諾:
  - 不替你定義一生
  - 不把生日變成判決
  - 不把數字變成階級
  - 不把象徵語言偽裝成絕對真理
  - 給你一個可以暫時借用的觀察角度
  - 讓你比較能感覺自己此刻怎麼動
```

## 6. 我們的姿態

新館的姿態是：

```yaml
溫柔:
  讓人願意靠近。

清醒:
  不神化，不誇大，不裝成命運權威。

好玩:
  保留遊樂園感，不把自我探索變成功課。

謙遜:
  承認這只是一種觀看方式。

不捕獲:
  不把人鎖進一個身份、數字或敘事。
```

### 6.1 說話方式

```yaml
推薦語法:
  - 你此刻可能比較靠近...
  - 這個數字可以被借來觀察...
  - 如果這句話有共鳴，可以看看...
  - 若不符合你的體感，可以先放下...
  - 這不是你的全部，只是一面鏡子。

避免語法:
  - 你天生就是...
  - 你注定會...
  - 你一定要...
  - 你的人生使命就是...
  - 你比其他數字更高階...
  - 這就是你的真實本質...
```

### 6.2 核心護欄句

```yaml
不是:
  你是幾號？

而是:
  這個數字如何幫你觀察當下的運動？
```

## 7. 八角框架的使用方式

本文件同意引入八角框架，
但只把它當作體驗設計檢查表，
不把它當成操控使用者的成長漏斗。

```yaml
定位:
  八角框架是外圈。
  協議姿態是內核。

目的:
  幫助我們檢查新館是否:
    - 有入口吸引力
    - 有探索節奏
    - 有回訪理由
    - 有可帶走的價值

限制:
  不用黑帽設計製造焦慮。
  不用損失感逼人回來。
  不用排行榜製造高低。
```

### 7.1 馴化後的八角檢查表

| 八角動力 | 新館可以怎麼用 | 必須避免 |
|---|---|---|
| 意義感 | 讓使用者感覺這是在觀察自己，而不是隨機娛樂 | 把結果包裝成宇宙判決 |
| 成就感 | 讓使用者感到「我更懂自己一點」 | 分數、等級、誰比較高 |
| 創造與選擇 | 讓使用者選擇看生日、今日、關係、工作或內在 | 塞滿選項導致焦慮 |
| 擁有感 | 允許本地保存自己的觀察筆記 | 強迫登入或資料收集 |
| 社會連結 | 可以分享一張觀察卡 | 比較誰的數字好、誰更稀有 |
| 稀缺性 | 少量使用季節性或每日卡片 | 製造錯過恐懼 |
| 驚喜感 | 每次回來有不同提問或切面 | 隨機到讓人依賴抽籤 |
| 損失避免 | 幾乎不用；只保留溫和提醒 | 連續登入、斷簽、錯過懲罰 |

### 7.2 黑帽設計禁區

```yaml
禁止:
  - 今日不看就錯過訊息
  - 連續簽到中斷懲罰
  - 稀有命格炫耀
  - 高階數字排名
  - 付費解鎖你的真正命運
  - 用焦慮推動分享
  - 用恐懼推動回訪
```

新館可以讓人想回來，
但不能讓人怕不回來。

## 8. 第一版體驗迴路

v0.1 的第一版網站體驗應該非常小。

```yaml
第一屏:
  - 生命靈數館名稱
  - 一句姿態句
  - 生日輸入
  - 進入按鈕

計算後:
  - 顯示一個主數字
  - 顯示這個數字的運動語言
  - 顯示一個今日觀察問句
  - 顯示「這不是你的全部」護欄

延伸:
  - 看 1～9 全表
  - 看更深的三段旅程
  - 進歷史區玩舊館小工具

離開:
  - 給一張可帶走的小卡
  - 不要求註冊
  - 不要求連續回訪
```

### 8.1 第一版不要做太多

```yaml
v0.1 不做:
  - 帳號系統
  - 排行榜
  - 社群比較
  - 大型個人報告
  - 付費命運解鎖
  - 複雜歷史紀錄
  - 完整天賦金字塔搬運
```

第一版只要完成一件事：

> 讓人第一次理解，生命靈數館不是告訴你「你是誰」，而是幫你觀察「你此刻怎麼動」。

## 9. 內容模型初稿

第一版每個數字可以先只有四個欄位：

```yaml
number:
  label:
    數字名稱

  movement:
    這個數字代表的 Δ 運動

  risk:
    容易卡住的方式

  question:
    今日觀察問句
```

範例：

```yaml
1:
  label: 起點差
  movement: 有一個差剛剛出現，正在等待展開。
  risk: 卡在「我要開始」，但還沒有真正進入關係或行動。
  question: 今天有哪個小小的開始，正在等你承認它存在？
```

這比一開始就做完整報告更適合新館。

## 10. 舊館與新館的關係

依照 [舊生命靈數館架構盤點](old-site-architecture-review.md)，
舊站應放進歷史區，而不是直接變成新館。

```yaml
舊館:
  角色:
    - 歷史小工具
    - 材料庫
    - 早期實驗紀錄

  可保留:
    - 天賦金字塔
    - 事業碼 / 家庭碼 / 坐鎮碼
    - 起因 / 過程 / 結果
    - 27 文本智慧架構
    - 手工解析 archive

  不直接繼承:
    - 舊 UI
    - 舊語氣
    - 靈魂使命式判定
    - 大型深度報告壓力

新館:
  角色:
    - 重新出生的公開入口
    - 更清醒、更輕、更不判決
    - 未來可逐步與協議語言對齊
```

## 13. 驗收標準

當新館 v0.1 做出來時，
可以用這份清單檢查：

```yaml
如果使用者看完結果後更想把自己定型:
  失敗

如果使用者看完後比較能觀察自己此刻怎麼動:
  成功

如果介面讓人焦慮、怕錯過、怕不回來:
  失敗

如果介面讓人感到可以回來，但也可以自由離開:
  成功

如果數字被做成高低階級:
  失敗

如果數字被做成觀察鏡頭:
  成功
```
