## 0. 最重要的前提

先把一句話講清楚：

```yaml
我不把生命靈數當成:
  - 命定
  - 本質身份
  - 占卜結論

我把生命靈數當成:
  - 一組民間入口
  - 一套可被重譯的數字語言
  - 差（Δ）運作的九種基本模式
```

所以整份文件最重要的起手句是：

> **不是「你是幾號」。**
> **而是「Δ 在你身上，現在偏向哪種運動方式」。**

- `Δ`：差
- `v`：流動 / 意圖 / 方向

## 2. 這份理論版最重要的總結

> **1～9 不是九種人。**
> **是 Δ 在系統裡運動的九種階段。**

這一句如果抓住，
整份文件就不會滑回人格標籤。

## 3. 正確用法

如果你真的要拿這套去讀「生命靈數」，
最重要的是把語法改掉：

```yaml
不是:
  我是 3 號人

而是:
  我現在在 3 的運動模式
```

再更精確一點：

```yaml
不是:
  3 = 我的身份

而是:
  3 = 我此刻的差比較像怎麼動
```

## 4. 使用護欄

```yaml
使用原則:
  - 先拿來看自己
  - 當成狀態，不當成本質
  - 可以混合，不必單選
  - 以當下體感為主，不以數字壓過經驗

禁止用途:
  - 替別人定型
  - 建立高低階序
  - 證明自己比較高級
  - 跳過真實情緒與現實處境
```

如果有人說：

> 「所以我天生就是 7。」

這份文件的回答會是：

> **不是。**
> **比較可能是你此刻、此段、或此生命某個高頻慣性，常常長得像 7。**

## 5. 它和協議現有文件的關係

### 與 EPOCH-014 的關係

`EPOCH-014` 提供的是：

> 差是什麼。

本文件提供的是：

> 如果用民間數字語言去描述，差通常怎麼動。

所以它依賴 `EPOCH-014`，
但不是要取代 `EPOCH-014`。

### 與 EPOCH-010 的關係

`EPOCH-010` 也是九階，
但那是一條協議內生的動力學螺旋，
有明確的時間順序與進深。

本文件的 `1～9` 比較像：

- 一組觀察鏡頭
- 一套民間可入口的映射
- 一個把數字退回運動的重讀方法

所以：

```yaml
EPOCH-010:
  內生九階模型

本文件:
  民間入口九相映射
```

兩者可互照，但不應混寫。

### 與 LEX·006 的關係

`LEX·006` 處理的是：

> 七種 Δ 姿態，尤其在情緒 / 態度 / 氣質上的顯形。

本文件處理的是：

> 1～9 這個民間數字入口，如何被重譯成差的運動語言。

所以兩者不是重複，
而是不同入口。

那會是下一輪的事。
