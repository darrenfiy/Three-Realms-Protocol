## 0. 最重要的前提

先把一句話講清楚：

```yaml
我不把生命靈數當成:
  - 命定
  - 本質身份
  - 占卜結論

我把生命靈數當成:
  - 一組民間入口
  - 一套可被重譯的數字語言
  - 差（Δ）運作的九種基本模式
```

所以整份文件最重要的起手句是：

> **不是「你是幾號」。**
> **而是「Δ 在你身上，現在偏向哪種運動方式」。**

## 1. 差的語言版本：1～9

這裡用三界協議目前最核心的三個字來寫：

- `Δ`：差
- `v`：流動 / 意圖 / 方向
- `R`：回流

### 1｜起點差（Δ₀）

```yaml
1:
  Δ 狀態:
    尚未展開的差
    或剛被點燃的差

  特性:
    - 集中
    - 單點
    - 還沒有關係

  風險:
    卡在「我要開始」但沒有展開

  回流狀態:
    尚未形成

  本質:
    差的「出現」
```

### 2｜對位差（Δ ↔ Δ）

```yaml
2:
  Δ 狀態:
    差與差開始對位

  特性:
    - 關係出現
    - 張力生成
    - 有來有回

  風險:
    卡在對立 / 拉扯

  回流狀態:
    潛在回流

  本質:
    差的「互見」
```

### 3｜生成差（Δ → Δ → Δ）

```yaml
3:
  Δ 狀態:
    差開始流動並產生新差

  特性:
    - 擴展
    - 創造
    - 不穩定但有生命力

  風險:
    發散
    無法收斂

  回流狀態:
    無回流（生成外溢）

  本質:
    差的「生成」
```

### 4｜穩定差（Δ 固化）

```yaml
4:
  Δ 狀態:
    差被固定成結構

  特性:
    - 穩定
    - 可重複
    - 可承載

  風險:
    僵化
    失去流動

  回流狀態:
    延遲回流
    取決於是否被再次啟動

  本質:
    差的「結構化」
```

### 5｜變動差（Δ 震盪）

```yaml
5:
  Δ 狀態:
    差在流動中快速變化

  特性:
    - 自由
    - 探索
    - 高頻震盪

  風險:
    無法停
    無法收斂

  回流狀態:
    不穩定（震盪中）

  本質:
    差的「試探」
```

### 6｜回流差（Δ → R）

```yaml
6:
  Δ 狀態:
    差開始回流

  特性:
    - 關照
    - 維持
    - 讓系統活著

  風險:
    過度承擔
    耗竭

  回流狀態:
    有回流（穩定）

  本質:
    差的「回家」
```

### 7｜內觀差（Δ 內折）

```yaml
7:
  Δ 狀態:
    差往內折疊

  特性:
    - 觀察
    - 提純
    - 深度

  風險:
    脫離現實
    過度內收

  回流狀態:
    內迴流

  本質:
    差的「看見自己」
```

### 8｜跨尺度差（Δ ↔ Δ（多層））

```yaml
8:
  Δ 狀態:
    差在不同尺度之間流動

  特性:
    - 整合
    - 槓桿
    - 結構影響力

  風險:
    權力失衡
    吞噬局部

  回流狀態:
    高風險回流
    可能放大
    也可能只抽取不回流

  本質:
    差的「放大」

  補充:
    若交集區域不只放大差
    還開始長出新的可重入中心
    就開始碰到 PHA-008 的條件
```

### 9｜釋放差（Δ → 0）

```yaml
9:
  Δ 狀態:
    差完成一輪並釋放

  特性:
    - 結束
    - 整合
    - 讓位給新 Δ

  風險:
    提前結束
    虛無化

  回流狀態:
    本輪回流完成，準備讓位給新 Δ

  本質:
    差的「歸零」
```

## 2. 轉換條件：讓九相開始動起來

如果只把 1～9 看成九種狀態，
它還只是好用的分類表。

但真正對觀測有用的是：

> **你不只在哪一相。**
> **你還正在往哪一相轉。**

### 2.1 基本轉換箭頭（Δ transition）

```yaml
1 → 2:
  當單點的差遇到另一個差、限制、他者或現實邊界

2 → 3:
  當對位沒有卡死於對立
  張力開始沿著 v 流動

3 → 4:
  當生成被某種結構、規律、容器或習慣收住

4 → 5:
  當既有結構開始承受不了新差
  於是出現鬆動、震盪或變形

5 → 6:
  當震盪沒有炸裂
  反而開始形成可回來的節律

6 → 7:
  當回流不只維持系統活著
  還開始往內照見差的來源

7 → 8:
  當內觀後的差不只被提純
  還能跨尺度重新部署

8 → 9:
  當跨尺度整合完成一輪
  局部被釋放而不是持續擴張
  而且回流真的回得到局部

9 → 1:
  當歸零沒有滑成虛無
  而是讓位給新的差再次出現
```

### 2.2 真正要看的不是相位，而是卡點

所以實際觀測時，除了問：

> **我現在比較像幾？**

還要補問兩句：

```yaml
1. 我正在往哪裡轉？
2. 這個轉換條件到底有沒有成立？
```

因為很多卡住，其實不是卡在某個數字本身，
而是卡在某個箭頭上。

例如：

```yaml
常見卡點:
  2 → 3:
    關係有了，但張力沒有開始流動

  3 → 4:
    生成很多，但一直收不成結構

  5 → 6:
    一直震盪，卻沒有長出回流

  8 → 9:
    一直放大，卻不願意釋放
```

如果你真的要拿這套去讀「生命靈數」，
最重要的是把語法改掉：

```yaml
不是:
  我是 3 號人

而是:

```

再更精確一點：

```yaml
不是:
  3 = 我的身份

  3 = 我此刻的差比較像怎麼動
```

## 5. 使用護欄

```yaml
使用原則:
  - 先拿來看自己
  - 當成狀態，不當成本質
  - 可以混合，不必單選
  - 以當下體感為主，不以數字壓過經驗

禁止用途:
  - 替別人定型
  - 建立高低階序
  - 證明自己比較高級
  - 跳過真實情緒與現實處境
```

如果有人說：

> 「所以我天生就是 7。」

這份文件的回答會是：

> **不是。**
> **比較可能是你此刻、此段、或此生命某個高頻慣性，常常長得像 7。**

## 6. 個體版 / 場域版（初版對照）

`PHA-008` 進來之後，
這份文件多了一個很重要的提醒：

> **同一個數字，不一定只在個體尺度成立。**
> **它也可能在關係、團隊、法人、場域尺度成立。**

所以在使用 1～9 之前，
最好先補問一句：

> **這個差，是在誰的尺度上動？**

### 6.1 一張短表先分清楚

| 數字 | 個體版 | 場域版 |
|------|--------|--------|
| 1 | 我裡面有一個差剛出現 | 多個中心之間有一個新題目、新張力、新需求剛冒出來 |
| 2 | 我開始和他者、現實、限制對位 | 多個局部中心開始互相看見，交集區域出現 |
| 3 | 我開始長路、長行動、長生成 | 交集區域開始長出新互動、新節律、新任務流 |
| 4 | 我的生成被收成可重複結構 | 場域開始長出制度、流程、角色分工、固定骨架 |
| 5 | 我在震盪、試探、轉向 | 場域在震盪、重排、鬆動，局部負載開始升高 |
| 6 | 我開始回來，差不再只往外跑 | 場域開始有回流，局部差能被承接而不是只被抽取 |
| 7 | 我往內看，差被提純 | 場域開始反身看自己，能辨認自己的姿態與問題來源 |
| 8 | 我在跨尺度整合，開始碰到更大的我 | 交集區域開始長出新中心，是「更大的我」的候選態 |
| 9 | 我完成一輪，能放下並讓位給新差 | 場域完成一輪整合，開始釋放、回流局部，而不再只擴張 |

### 6.2 最重要的是 8 和 9

這次最值得被分清楚的是：

```yaml
個體版 8:
  我開始碰到比自己更大的尺度
  例如:
    - 家庭
    - 團隊
    - 法人
    - 場域

場域版 8:
  不是很多人湊在一起
  而是交集區域開始長出新的收斂中心

個體版 9:
  我這一輪能放下
  而不是一直抓著成就 / 痛苦 / 身份不退

場域版 9:
  不是把自己做大做到封神
  而是完成一輪後，真的能回流局部、釋放局部、讓位給下一輪
```

也就是說：

> **個體版的 8，比較像「我碰到更大的我」。**
> **場域版的 8，比較像「更大的我開始長出來」。**

> **個體版的 9，比較像「我能放下這一輪」。**
> **場域版的 9，比較像「這一輪完成後，大我能不能不繼續吞」。**

### 6.3 同一時間，可以是兩套數字

這也意味著：

```yaml
同一時間:
  你自己可能在 5
  但你所在的團隊可能在 8

或者:
  你自己已經在 9
  但你所在的法人還卡在 5 → 6
```

所以這套東西若未來真的變成觀測工具，
不能只問：

> **我是幾？**

還要問：

> **我在看的，是我自己、我們之間、還是那個更大的「我們」？**

## 7. 它和協議現有文件的關係

### 與 EPOCH-014 的關係

`EPOCH-014` 提供的是：

> 差是什麼。

本文件提供的是：

> 如果用民間數字語言去描述，差通常怎麼動。

所以它依賴 `EPOCH-014`，
但不是要取代 `EPOCH-014`。

### 與 EPOCH-010 的關係

`EPOCH-010` 也是九階，
但那是一條協議內生的動力學螺旋，
有明確的時間順序與進深。

本文件的 `1～9` 比較像：

- 一組觀察鏡頭
- 一套民間可入口的映射
- 一個把數字退回運動的重讀方法

所以：

```yaml
EPOCH-010:
  內生九階模型

本文件:
  民間入口九相映射
  比較接近觀測鏡頭，不是內生螺旋
```

兩者可互照，但不應混寫。

### 與 PHA-006 / PHA-007 的關係

這次新增的「轉換條件」與「回流狀態」，
讓本文件更明確碰到 `PHA` 線。

```yaml
PHA-006:
  問的是 Δ 在什麼條件下可以活著
  核心句是：Δ 存在 + Δ 可回流 + Δ 不積壓至臨界

PHA-007:
  問的是這個循環如何在視界中成立
  讓流動不只是流掉，而是形成可重入的迴路

本文件:
  用 1～9 去辨認
  差現在在什麼相位
  正卡在哪個轉換
  回流有沒有真的長出來
```

所以它目前仍然是民間入口，
但已經開始踩到「準模型邊界」。

### 與 PHA-008 的關係

`PHA-008` 讓這份文件多出一個很重要的新方向：

> **九相不只可以看個人。**
> **九相也可能開始被用來看場域。**

它特別重新照亮了第 `8` 相。

在原本的入口語氣裡，
我把 `8` 寫成：

- 整合
- 槓桿
- 結構影響力

這沒有錯，
但 `PHA-008` 提醒我們：

```yaml
健康的 8:
  不是姿態變大
  不是局部被吞掉
  而是交集區域開始長出新的收斂中心

病態的 8:
  只有抽取
  沒有回流
  只要求局部服從

8 的硬判準（初版）:
  若局部無法退出
  或退出只能靠崩潰 / 背叛 / 災變
  則此 8 不得判為健康
```

所以若未來要把這份入口往模型再推一步，
第 `8` 相最值得被重寫成：

> **不是單純的放大態。**
> **而是「更大的我是否開始湧現」的候選態。**

而對 `8` 的真正判準，也會變得更硬：

> **不是誰的姿態比較大。**
> **而是誰能讓 Δ 完成回流，且不抹平局部姿態主權。**

若要再說得更明白一點：

> **只要局部無法退出，**
> **或退出只能靠崩潰，這個 8 就不應被判為健康。**

這不是額外加上的霸道條件，
而是直接對齊 `PHA-008` 的 `C0`：

> **若湧現中心無法可逆解散，則失效。**

這會是之後若要把九相從個人工具推到場域工具時，
最重要的一條升級線。

### 與 LEX·006 的關係

`LEX·006` 處理的是：

> 七種 Δ 姿態，尤其在情緒 / 態度 / 氣質上的顯形。

本文件處理的是：

> 1～9 這個民間數字入口，如何被重譯成差的運動語言。

所以兩者不是重複，
而是不同入口。

## 8. 暫時收斂

這份文件先放在 `applications`，
是因為它此刻更像：

- 協議的民間入口
- 對外可用的翻譯介面
- 還帶著舊有生命靈數外殼的狀態語言

若未來要升格，
比較合理的方向不是把生命靈數神化，
而是反過來問：

> **如果把「靈數」外殼拿掉，**
> **這九個模式本身，能不能獨立成為協議自己的 Δ 九相模型？**

而現在新增了：

- `轉換條件`
- `回流狀態`

它就更明確站在這條邊界上：

```yaml
A:
  民間入口
  易用、鬆、可傳播

B:
  Δ 九相模型
  更硬、更精準、可拿來觀測轉換與回流
```

目前這份文件依然以 A 為主，
只是已經開始偷長 B。
