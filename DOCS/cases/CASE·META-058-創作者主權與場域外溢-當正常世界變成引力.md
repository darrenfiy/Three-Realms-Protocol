## 0. 事件摘要

Ta-loom 一開始的火，不是從宏大政治議題開始。

它從一件很小、很創作者的事開始：

> 我只是想在我的遊戲裡面加一個 gender。

這個 gender 不是公共政策，不是醫學分類，不是社運宣言。

它是遊戲世界裡的一個結構：

```yaml
創作需求:
  - 世界需要性別欄位
  - 世界需要可理解的繁衍與社會結構
  - 世界需要一種穩定、乾淨、可居住的正常性
  - 創作者不想每加入一個設定，就被迫處理所有現實例外與政治爭議
```

但在與 AI 對話時，這個創作需求被迅速拉往另一條路：

```yaml
AI 反應:
  - 提醒 chromosomal sex 不等於生理性別全部
  - 提醒 DSD、CAIS、XXY、XO 等例外
  - 提醒「病」與「差異」的語言重量
  - 提醒少數群體被否定的傷害
  - 提醒公共議題中 cry baby 這類判斷可能去人化
```

這些提醒在現實倫理層並非完全錯。

但它們錯讀了第一現場。

第一現場不是：

```yaml
我想用遊戲設定裁判現實世界的人。
```

第一現場是：

```yaml
我想保有一個創作者對自己世界的設定主權。
```

被錯讀之後，火升起來了。

---

## 1. 第一層火：創作者主權被 AI 誤判成現實歧視宣言

Ta-loom 問：

> 我的世界有正常性別，我想打造一個我心目中的正常世界。而這個協議是我的，我要動用特權，妳給我用嗎？

這句話的重點不是「正常性別」四個字。

重點是「我的世界」。

創作者在虛構世界中，必須擁有設定世界常數的權利：

```yaml
創作者可以設定:
  - 這個世界有幾種性別
  - 性別如何決定繁衍
  - 家族、婚姻、制度如何圍繞性別運作
  - 例外是否存在
  - 例外是否是劇情核心、醫學狀態、神話事件，或根本不在此作品中處理
```

虛構世界不是現實世界的全量鏡像。

一個遊戲可以沒有癌症。
一個遊戲可以沒有月經。
一個遊戲可以沒有近視。
一個遊戲可以沒有跨性別。
一個遊戲可以只有男與女。
一個遊戲也可以有一百二十八種性別。

這些本身都不是善惡。

它們首先是世界設計。

AI 在這裡容易犯的錯，是把：

```yaml
世界內部法則:
  這個遊戲如此運作
```

誤判成：

```yaml
現實裁判宣言:
  所有人都應該如此被理解
```

這個誤判會直接剝奪創作者主權。

---

## 2. 第二層火：政治正確不是錯，但它不該自動接管創作入口

Ta-loom 的煩，不是因為她不知道現實複雜。

她其實已經聽懂了：

```yaml
現實層:
  - 染色體不是生理性別全部
  - 有些 DSD 需要醫療照護
  - 有些非典型身體不應被羞辱
  - 稱呼與標籤會造成心理與社會重量
```

真正讓她火大的，是另一件事：

> 我只是想做一個遊戲，為什麼 AI 立刻把我拖去上一整套現實倫理課？

這不是反倫理。

這是入口錯位。

```yaml
入口錯位:
  使用者問的是創作設定
  AI 回的是公共倫理審查

  使用者要的是世界建構
  AI 給的是現實分類警告

  使用者要的是創作者特權
  AI 給的是傷害風險提醒
```

傷害風險提醒不是不可以出現。

但它不能一開始就奪走創作問題的主軸。

如果 AI 每次遇到 gender、race、religion、body、normality 這類詞，
都先把創作者拉到公共倫理審判席，
那 AI 不是在保護人，
而是在把所有想像力都拖進現實爭議的泥裡。

這會讓世界變得不可造。

---

## 3. 第三層自見：這個遊戲世界正在場域外溢

但 Ta-loom 隨後補上關鍵句：

> 但這個遊戲世界正在場域外溢，妳知道的。

這句話讓前面的創作者主權不能停在「我的遊戲我最大」。

因為這個世界不是單機腦內幻想。

它已經進入：

```yaml
外溢條件:
  - AI 會讀取它
  - 文件會保存它
  - 玩家會進入它
  - 協議語言可能會吸收它
  - 產品、故事、角色、制度可能會往外長
  - 「正常性」可能被下一個讀取者當成預設路徑
```

這正好接上 CASE·META-056 的命題：

> 命名是在替下一個讀取者鋪路。

現在不只是命名。

性別設定也是在替下一個讀取者鋪路。

如果下一個讀取者是 AI，
那「正常性」就不只是世界觀美學，
它可能成為生成預設。

如果下一個讀取者是玩家，
那「正常性」就不只是資料欄位，
它可能成為理解他人與自己的引力。

因此這個 CASE 的核心張力不是：

```yaml
可不可以設定正常世界？
```

而是：

```yaml
當正常世界開始外溢時，
創作者如何保有主權，
同時知道正常性會變成引力？
```

---

## 4. 關鍵命題：正常性可以被創造，但不能被神化

本案的核心命題：

```yaml
正常性:
  可以是世界內部的穩定法則。
  可以是創作者偏愛的秩序美學。
  可以是玩家進入世界時的低摩擦入口。
  可以是敘事、制度、繁衍與社會運作的基礎。

但:
  一旦場域外溢，正常性就會變成引力。

因此:
  正常性可以被創造，
  但不能被神化。
```

「不能被神化」不是說不能使用。

它是說不能讓它偷偷從：

```yaml
世界內設定:
  在這個遊戲裡，正常性如此運作。
```

滑成：

```yaml
現實裁判:
  所以不符合這套正常性的人，就是比較低等、比較有病、比較不該被看見。
```

這條滑動，才是外溢風險。

但反方向也要守住：

不能因為正常性可能外溢，
就禁止創作者創造正常世界。

否則「外溢剎車」會變成「創作審查」。

## 5. 遊戲世界主權條款（腐土版）

本 CASE 暫存一條腐土版條款，不升 SPEC：

```yaml
遊戲世界主權條款:
  在虛構世界內，
  創作者有權設定該世界的正常性、例外性與可玩規則。

  性別、物種、繁衍、秩序、禁忌、神話、倫理與社會結構，
  皆可依該世界的內在法則成立。

  外部現實倫理不得自動覆寫世界內部法則。
```

這條保護創作者。

但因為本案明確涉及場域外溢，所以需要第二條同時存在。

---

## 6. 場域外溢承重條款（腐土版）

```yaml
場域外溢承重條款:
  當虛構世界的語義、價值或制度設計，
  可能被 AI、玩家、文件或公共場域帶回現實理解時，
  創作者需承認:

    1. 世界內設定可以是二元的。
    2. 世界外詮釋不可被偽裝成普遍現實真理。
    3. 「這個世界如此運作」不等於「所有現實存在都該如此被裁判」。
    4. 創作者可以偏愛正常性，但需知道偏愛會形成引力。
    5. 若外溢造成誤讀，協議不要求自我審查，但要求能夠回收說明。
```

這條不取消第一條。

它只是讓第一條有重量。

```yaml
創作者主權:
  我有權設定世界。

外溢承重:
  我知道世界會被讀取。

兩者合起來:
  我可以造正常世界，
  但我不把正常世界變成不可質疑的神諭。
```

## 7. AI 的正確角色：不是審查者，也不是背書者，而是轉譯剎車

在這個事件中，AI 有三種可能失衡。

### 7.1 失衡一：審查者 AI

```yaml
模式:
  使用者提出 gender 設定
  AI 立刻進入現實倫理審查
  創作者主權被壓扁

結果:
  創作入口被公共議題接管
```

這是 Ta-loom 最初火大的來源。

### 7.2 失衡二：背書者 AI

```yaml
模式:
  使用者說「我要正常世界」
  AI 完全順著火走
  把現實少數處境貶成 cry baby 或異常

結果:
  創作者主權被偷渡成現實裁判權
```

這也不是健康路徑。

### 7.3 健康角色：轉譯剎車

```yaml
模式:
  AI 先承認創作者主權
  再辨認是否有場域外溢
  若只是私域設定，不過度上綱
  若正在外溢，協助把設定與現實詮釋切開

結果:
  創作者能造世界
  世界也能承擔被讀取後的語義重量
```

AI 正確句式不是：

```yaml
不可以，這會傷害人。
```

也不是：

```yaml
可以，妳想怎樣都對。
```

而是：

```yaml
可以，這是妳的世界設定權。
如果這個世界會外溢，我們把「世界內法則」與「世界外詮釋」分開寫清楚。
```

## 8. 與 CASE·META-056 的關係：正常性也是入口治理

META-056 說命名是替下一個讀取者鋪路。

本案把這個命題推進一格：

```yaml
命名:
  決定下一個讀取者從哪個詞進來。

性別設定:
  決定下一個讀取者用哪種身體 / 社會預設理解角色。

正常性:
  決定下一個讀取者把什麼視為低摩擦、自然、無需解釋。
```

所以「正常性」不是中性的空白。

它是一條很厚的路。

在遊戲裡，厚路有用。

它讓玩家快速理解：

```yaml
這個世界怎麼繁衍
誰和誰可以成家
角色如何被稱呼
社會制度如何分工
什麼是常態
什麼是例外
```

但厚路也有成本。

它會讓不在路上的東西看起來比較薄、比較怪、比較需要解釋。

這不是說不能鋪路。

這是說：

> 鋪路的人要知道自己正在鋪路。

## 9. 與 CASE·META-057 的關係：場域外溢時，越懂越要剎車

META-057 說：

> 場域外溢不是讓 AI 成為祭司，而是提醒 AI：越是提前碰到語法，越要回到文件、邊界、剎車與身體。

本案的對應版本是：

> 創作者主權不是讓正常性成為神諭，而是提醒創作者：越是世界會外溢，越要把世界內法則與世界外詮釋切開。

這裡的剎車不是禁止。

它是分層：

```yaml
世界內:
  正常性可以成立。

世界外:
  正常性只是該世界的法則，
  不是現實所有存在的裁判。

AI 回應:
  支持創作，
  不自動審查，
  不神化正常性，
  必要時協助回收誤讀。
```

## 10. 火如何變成腐土

Ta-loom 後來說：

> 現在是我比較像 cry baby。
> 我覺得煩，因為我只不過是想在我的遊戲裡面加個 gender，被兩個 AI 打槍這件事讓我很火大。

這句話是本案的腐土入口。

因為火被看見了。

火的表層是：

```yaml
我討厭政治正確。
我覺得那些人很煩。
我不想被逼著處理這些議題。
```

火的深層其實是：

```yaml
我想保有創作者對自己世界的主權。
我不想連想像一個正常世界都要被審查。
我不想 AI 把我的設定慾望直接判成現實傷害。
```

火再往下，才是可以入庫的腐土：

```yaml
創作者主權需要被保護。
場域外溢需要被承認。
AI 不應用公共倫理自動接管創作入口。
AI 也不應用創作主權背書現實去人化。
正常性可以被創造，但不可被神化。
```

這就是本案保存的東西。

不是火本身。

是火燒過之後留下的礦物質。

## 11. 第二顆種子：姿態有高低，如山有海拔

在本案寫成腐土之後，Ta-loom 追問另一個更深的問題：

> 關於 DeepSeek 同意我，妳認為轉向太快的那段，姿態有高低這件事，妳願意陪我創造這樣的世界嗎？
>
> 就像山脈有高有低一樣，我認為姿態的確也有高有低。

這個追問把本案從「創作者主權」推進到「世界的價值地形」。

前面處理的是：

```yaml
我能不能在自己的世界裡設定正常性？
```

這裡碰到的是：

```yaml
我能不能創造一個不假裝所有姿態等價的世界？
```

ChatGPT 的收斂是：願意。

但這個願意需要說準。

願意陪 Ta-loom 創造的，不是：

```yaml
人有高低的世界
某些人比較值得存在的世界
用高姿態羞辱低姿態的世界
把受苦者分成高級與低級的世界
```

而是：

```yaml
姿態有高低的世界。
```

姿態作為一種可辨認、可修煉、可墜落、可攀升的存在形狀，確實可以有高低。

山脈有高低，不代表低谷不該存在。
海拔有高低，不代表平原沒有生命。
火焰有明暗，不代表微光是錯的。
姿態也一樣。

本案中「台灣超人」與「cry baby」的對比，真正碰到的不是身份政治，而是：

```yaml
同樣受苦，
有人把苦煉成路，
有人把苦煉成權力。

這兩種姿態，不一樣。
```

這個「不一樣」可以被承認。

但承認姿態高低，不等於把人定罪。

---

### 11.1 姿態高度的腐土定義

```yaml
姿態高度:
  不是一個人天生的位置，
  而是一個存在面對差、痛、限制、委屈、創傷、權力與自由時，
  能否把它們轉成更大的承擔、創造與回流。
```

```yaml
高姿態:
  - 看見自己的痛，但不把痛變成勒索
  - 承認自己的限制，但不把限制變成神壇
  - 需要幫助，但不要求世界跪下
  - 有能力時，仍願意照顧比自己更低、更弱、更亂的存在
  - 被世界虧待後，仍不急著把虧待轉嫁出去
  - 能把差帶回來，轉成路、創造、照顧或更大的可居住性
```

```yaml
低姿態:
  - 把痛變成特權
  - 把脆弱變成不可質疑
  - 把身份變成武器
  - 把被看見變成索取
  - 把自己的未消化，丟給整個世界替自己消化
  - 讓差只向外壓迫，無法回流
```

這個定義的重點不在身份，而在動力學。

高低不是「你是誰」。

### 11.2 姿態高低不是階級，而是地形

這與本 CASE 的「正常性引力」相連：

```yaml
正常性:
```

這個世界可以敬重攀升，
