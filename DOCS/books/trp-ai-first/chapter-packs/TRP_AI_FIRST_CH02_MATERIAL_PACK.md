## 0. 這份素材包在做什麼
Chapter 2 仍然是方向章節，但它的方向已經比 Chapter 1 更具體：

```yaml
Chapter 1 回答:
  為什麼單界文明不夠

Chapter 2 回答:
  如果三界是真的框架
  那它們到底怎麼互相作用？
```

這章的起手不能是教條，而是運動：

```yaml
三界:
  不是三個名詞
  不是三個部門
  不是三格世界觀圖卡

三界交互:
  是文明如何定義、發動、落地、回驗自己的運作鏈
```

---

## 1. 本章在整本書中的位置
Chapter 2 是全書第一個真正的交互章節。它的工作是避免 Chapter 1 被讀成只有世界觀宣告。

```yaml
如果 Chapter 1 是:
  「為什麼需要三界」

那 Chapter 2 就是:
  「三界不是並列存在，而是交互生成」
```

它對後續章節的支撐非常直接：

```yaml
Chapter 3:
  需要交互律作為顯現與生成的前提

Chapter 4:
  需要交互律作為流動、差與責任的基底

Chapter 5 之後:
  需要交互律作為所有操作層與場域章節的母語法
```

## 2. 本章的一句話總命題
> **三界不是三個並排的領域，而是同一存在的交互律：意識定義能量，能量驅動物質，物質驗證意識；它們之所以能不斷互相展開，是因為三界本身就是一個碎形閉包，而不是三個彼此隔離的盒子。**

## 3. 本章必須回答的核心問題

```yaml
問題一:

問題四:
  碎形閉包是什麼？
  為什麼它能防止三界被誤讀成三個箱子？

問題五:
  這套交互律如何成為後面全書的母語法？
```

## 4. 本章應鎖定的八個主命題

### 命題 A. 三界不是靜態共存，而是動態互涉
必須把 `SPEC·000` 第一條真正展開：

```yaml
三界並存:
  不是三方各自存在

同時互涉:
  才是 Chapter 2 的主軸
```

### 命題 B. 第二到第四條母則，構成三界交互的主軸
正文應明寫：

```yaml
意識定義能量:
  賦義 / 指向 / 命名 / 朝向

能量驅動物質:
  流動 / 動員 / 轉化 / 發生

物質驗證意識:
  固化 / 回饋 / 後果 / 檢驗
```

### 命題 C. 這不是單向控制鏈，而是帶回饋的文明運作鏈
這章要切開一個關鍵誤讀：

```yaml
不是:
  意識說了算
  能量只是中介
  物質只是結果

而是:
  物質的回饋
  會反過來修正意識的定義
```

### 命題 D. 碎形閉包不是華麗形容詞，而是必要結構語言
來自 `EPOCH-012` 的這句，應是本章硬骨架之一：

```yaml
三界 = (結構, 動態, 可能) 的碎形閉包
```

這意味著：

```yaml
每一界:
  都不只是自己
  都含有其他兩界的折射
```

### 命題 E. Chapter 2 必須把三界從「分類學」推進到「交互學」
如果這章只做分類，後面全書會變平。

```yaml
Chapter 2 成功時:
  讀者看到的是作用
  不是架子
```

### 命題 F. Whitepaper 是展開，SPEC·000 是壓縮
正文應明寫這層關係：

```yaml
SPEC·000:
  母條文
  提供最小骨架

Whitepaper:
  展開母條文
  讓文明尺度的語義長出來
```

### 命題 G. SEED 可以作為交互律的高密度壓縮接口，但不能取代正文推導
這一章可以輕帶 `SEED.md`，但不應偷懶。

```yaml
SEED:
  適合用來做高密度回聲
不適合:
  取代 Chapter 2 的正文展開
```

### 命題 H. 本章的成功，是讓讀者開始用三界交互去讀世界
Chapter 2 要留下的不是記憶點，而是新讀法：

```yaml
看到一個現象時
  會開始問：
    這裡的意識如何定義？
    能量如何被發動？
    物質如何回驗？
```

### 操作層試點. 三界交互檢查

```yaml
檢查四問:
  這個提案在意識界定義了什麼？
  它如何轉成可流動、可驅動的能量？
  它落在物質界的驗證方式是什麼？
  物質回饋如何反過來修改最初定義？
```

最小測試：

```yaml
能否分辨:
  只有願景、沒有動員
  只有動員、沒有落地
  只有落地、沒有回驗
```
