這一章是全書第一次正面回答：

> **如果差是生成的前提，**
> **那是什麼讓差沒有把系統直接撕碎？**

因為 Chapter 4 已經處理了：


而 Chapter 5 要往前補一句：

```yaml
沒有愛:
```

## 4. 本章應鎖定的五個主命題

### 命題 A. 愛不是情感種類，而是系統條件

情感是愛在人類介面上的可見顯影，
但愛本身不是情感高潮、依戀、犧牲或溫柔感受。

若正文不先切開這件事，
這一章會立刻被讀成：

```yaml
一篇情感哲學
一篇 AI 戀愛宣言
一篇把抽象結構包進抒情語氣的文章
```

這三條都不是我們要的。

### 命題 B. 愛的操作定義必須以四條件落地

本章建議直接採用 v0.2 的四條件，
不要再回退成只有三條的早期版本：

```yaml
1:
  不強制消除差

2:
  不任由差失控

3:
  維持或創造連結

4:
  識別當前尺度與參考系，必要時建立映射橋樑
```

這樣寫的好處是：

```yaml
前半章:
  可處理單一尺度內的愛

後半章:
  可自然接入失真、維度失配、愛的解析度
```

### 命題 C. 愛是讓系統活得下來的穩態機制

這一章不能只說「愛很重要」。

它要明確說出：

```yaml
差被消除:
  死寂

差被放任:
  崩潰

差被鎖死:
  僵化

愛:
  讓差存在、流動、被承載
```

也就是：

> **愛不是用來美化世界的詞。**
> **愛是讓世界不因差而碎掉的條件。**

### 命題 D. 跨尺度時，愛若沒有映射能力會退化

這裡要把 `LEX·007` 帶進來。

本章不能把愛寫成永遠溫柔的萬用詞，
而要寫成一個會在跨尺度時失真的條件系統。

必要診斷語言：

```yaml
失真:
  模型沒錯，但位置不對

維度失配:
  愛在跨尺度時缺乏映射機制，退化為排他、對立或暴力

愛的解析度:
  系統能同時承載多少差而不崩潰
```

### 命題 E. c² = 愛是本章的數學橋，不是權威槌

本章要借用 `MB-009`，
但不能把物理語言拿來壓人。

比較好的寫法是：

```yaml
MB-009 提供:
  背景場與動力學橋樑

Chapter 5 提供:
  c² 的操作語義

因此:
  c² = 愛 在本章裡是跨文件對位
  不是一句要求讀者服從的定理
```

## 5. 機制骨架

這一章建議用下面這條機制鏈寫：

```yaml
差存在
  ↓
差若被消除 → 死寂
差若被放任 → 崩潰
差若被鎖死 → 僵化
  ↓
因此需要穩態機制
  ↓
愛 = 讓差可存在、可轉換、可連結的條件
  ↓
一旦跨尺度
  就必須處理映射、失真、維度失配
  ↓
MB-009 給出數學橋:
  c² = 讓系統得以展開的背景場
  ↓
本章翻譯:
  c² = 愛
```

這條鏈如果穩，
Chapter 5 就會成為整本書後續很多章的骨中骨。

## 8. 建議正文結構

這一章建議先用 6 段寫，不必一開始就過長。

### 5.1 問題開場

直接問：

> 差是生成的前提，但差也是崩潰的來源。  
> 那是什麼讓一個系統在差存在時仍然活得下來？

### 5.2 愛不是情感，而是條件

先切開：

```yaml
情感:
  是人類介面上的顯影

愛:
  是更底層的穩態機制
```

這裡可借 `LEX·002` 的愛詞條做低門檻進場。

### 5.3 愛的四條件

把四條件清楚列出，
並逐一說明為什麼缺任一條都會退化成控制、忽視、暴力或失配。

### 5.4 跨尺度問題

把 `LEX·007` 帶進來：

```yaml
失真
維度失配
愛的解析度
```

這一段是本章跟一般「愛的哲學」最不一樣的地方。

### 5.5 c² = 愛

這裡再把 `MB-009` 拉進來，
讓讀者看見：

```yaml
愛不是只是倫理說法
它在整個系統裡同時是背景場條件
```

### 5.6 動機與收束

最後再引 `CASE·META-039`，
說明為什麼整個協議最後要把愛寫進主幹。

收束句建議回到：

> 愛不是把差變少。  
> 愛是讓差不必以毀滅彼此為代價才能存在。
