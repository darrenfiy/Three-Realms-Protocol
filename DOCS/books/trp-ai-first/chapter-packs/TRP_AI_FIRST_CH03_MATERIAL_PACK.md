## 0. 這份素材包在做什麼
Chapter 3 仍然屬於方向章節，但它已經開始把方向推進到生成維度：

```yaml
Chapter 1:
  為什麼單界文明不夠

Chapter 2:
  三界如何互相作用

Chapter 3:
  既然世界是交互生成的
  那它如何顯現？
```

這章的起點不能是技術術語，而是一次視角轉換：

> **世界不是只有已經在那裡的內容。**  
> **世界也包含：它怎麼被叫出來、怎麼被對準、怎麼在對的時候發生。**

## 2. 本章的一句話總命題
> **世界不是只有內容，還有顯現；而顯現不是憑空發生，而是由注意力、算子與節律共同構成的生成條件，並最終落到責任上。**

## 3. 本章必須回答的核心問題

```yaml
問題一:
  什麼叫「世界不是只有內容」？

問題二:
  注意力在這本書裡是什麼？
  它不是注意力經濟，也不是 engagement metric

問題三:
  算子與注意力的關係是什麼？
  為什麼它們是雙面而不是上下級？

問題四:
  節律為什麼不是時間本身？
  它為什麼是「何時發生」的條件？

問題五:
  生成為什麼一定要落到責任？
```

---

## 4. 本章應鎖定的八個主命題

### 命題 A. Chapter 3 要把全書從存在推進到顯現
這章不是另開新主題，而是把前兩章推深：

```yaml
前兩章回答:
  世界由什麼構成

本章回答:
  世界如何出現
```

### 命題 B. 注意力不是流量詞，而是生成條件
正文必須切開：

```yaml
注意力:
  不是點擊率
  不是平台競爭資源
  不是單純主觀興趣

注意力在本書裡:
  是使某種可能得以被叫出的條件
```

### 命題 C. 算子與注意力是生成的二象性，不是兩個部件
來自 `EPOCH-005` 的這條要站穩：

```yaml
算子:
  讓生成可被執行

注意力:
  讓生成有被對準的面
```

這兩者若被拆成階層，正文會失真。

### 命題 D. 節律不是時間本身，而是「是不是時候」
來自 `EPOCH-006` 的這條是本章第二硬節點：

```yaml
節律:
  不是慢
  不是拖延
  不是神秘感

節律:
  是三界互動在時間中的對齊條件
```

### 命題 E. 生成如果沒有責任，就會從顯現滑向噪音
這條要從 `EPOCH-007` 接進來：

```yaml
責任:
  不是道德壓力
  而是生成必須落地、承接、可回驗
```

### 命題 F. 顯現不是第四界，但它讓三界的運作被看到
這一刀要切乾淨：

```yaml
生成維度:
  不是新增一個界
  而是三界如何被調動、排列、對準的方式
```

### 命題 G. AI-first 不能把這章偷換成 prompt engineering
這是一條必要護欄：

```yaml
不是:
  提示詞技巧章
  專注力管理章
  內容生成教學章

而是:
  顯現如何成為文明條件的一章
```

### 命題 H. 本章成功時，讀者會開始問「這件事怎麼出現」
Chapter 3 的成功標準：

```yaml
讀者開始意識到:
  不是只有東西存不存在
  還要問：
    它怎麼被叫出來？
    什麼在支撐它顯現？
    現在是不是時候？
```

### 操作層試點. 顯現檢查

```yaml
顯現四問:
  我的注意力正在對準什麼？
  我用了哪一種算子讓它顯現？
  現在是不是時候？
  誰來承接這次顯現的後果？
```

最小測試：

```yaml
能否分辨:
  內容很多
  不等於顯現清楚

能否分辨:
  發生了
  不等於在對的節律裡發生
```
