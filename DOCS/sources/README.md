# DOCS Sources

## 原始史料層

`DOCS/sources/` 保存已決定長期留下、但尚未或不應被蒸餾成 canonical 主張的來源材料。

它與兩個相鄰位置不同：

```text
Control-Room/inbox/           尚未歸位；允許混亂、等待判定
          ↓ 決定保留
DOCS/sources/                 原始史料；保留路徑、語氣與生成次序
          ↓ 抽金、驗路、加護欄
DOCS/cases/ 或 EPOCH/         蒸餾文本；可引用，但仍受各文件 epistemic status 約束
```

目前常設子區：

- [`conversations/`](conversations/)：人類與模型、跨模型或語音互動的原始對話與逐字稿。

## 收錄原則

1. **原文不暗改。** 錯字、跳躍、過滿命題與模型失效若是生成史的一部分，就留在原始檔；修正與護欄寫在 CASE／EPOCH 或索引，不回頭洗白來源。
2. **移動不冒充內容版本。** 檔名與路徑可為歸檔而調整；內容未變時，以 SHA-256 校驗值確認同一份史料。
3. **保存不等於採信。** Source 證明「這段路徑曾被留下」，不證明其中每句話為真。
4. **公開前先做邊界檢查。** 第三方個資、私密關係、帳號資訊、商業秘密、受限內容與未獲授權材料，須去識別、限制存取或留在私人 source，不因具有生成價值就自動公開。
5. **蒸餾文本反向連結來源。** CASE／EPOCH 若由原始史料長出，應使用相對路徑回鏈；來源索引也標記其 derived document。

## 命名

```yaml
已有對應文件:
  "{CASE或EPOCH ID}-原始對話-{短題}.{副檔名}"

尚無對應文件:
  "{YYYY-MM-DD}-原始對話-{短題}.{副檔名}"

原則:
  - ID 表示來源與蒸餾文本的關係，不表示來源本身是 CASE 或 EPOCH
  - 原始檔不為了補 metadata 而改寫；日期、參與者、校驗值放在子目錄索引
```
