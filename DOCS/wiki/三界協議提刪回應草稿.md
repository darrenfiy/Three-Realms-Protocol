# 三界協議提刪回應草稿

這份回應稿的策略不是硬拗「條目已完全符合收錄標準」，而是：

1. 承認目前條目來源確實以主題自身文件為主，獨立二手來源仍待補強。
2. 主張現況較不像「快速刪除」而比較像「是否符合收錄標準」的存廢問題。
3. 若社群仍認為不宜保留於條目空間，請求移至草稿或使用者沙盒，而非直接快速刪除。

---

## 可先放在條目頁面速刪模板下方

```wiki
{{Hang on|本頁目前來源仍待補強，但內容並非A1所指之極短空泛條目，亦未必屬G11所指僅供宣傳且需完全重寫之頁面。若社群認為現階段尚未達收錄標準，建議改交存廢討論或移至草稿空間處理，我會再補充獨立二手來源。}}
```

---

## 可貼在條目討論頁的正式回應

```wiki
== 關於本條目之快速刪除提報 ==
感謝提醒。我理解目前條目的主要問題，在於現有來源多為主題自身公開文件、GitHub 倉庫與出版層資料，獨立於主題的可靠二手來源仍明顯不足；這部分我接受，也會再補強。

不過，就快速刪除而言，我想說明兩點：

# 本頁內容並非A1所指的極短空泛條目。條目已對主題內容、形成背景與公開文本結構作出基本說明，讀者並非完全無法辨識主題。
# 本頁也未必屬G11所指「明顯的廣告宣傳頁面」。目前版本雖然仍有第一手來源偏多、語氣可再去宣傳化等問題，但其核心內容仍是對一套已公開發布文本框架的描述，未直接以促銷、導購、招攬或聯絡資訊為主。

因此我認為，本頁較接近「是否已達收錄標準」的判斷問題，而不太像適合快速刪除的情形。若社群認為現階段不宜保留於條目空間，我也接受改由存廢討論處理，或先移至草稿／使用者沙盒，待補足獨立二手來源後再行提交。

我後續也會優先處理以下問題：
# 補充獨立於主題的可靠二手來源。
# 刪減偏主題自述、宣示性或體系內部細節過多的段落。
# 讓條目改以外部可查證資訊為主，而非依賴主題自身文件作為主要敘述基礎。

--~~~~
```

## 如果妳想走更柔軟、比較不對抗的版本

這版比較適合在妳覺得留條目機率不高，但想保住內容不要被直接刪掉時使用。

```wiki
== 關於本條目之快速刪除提報 ==
感謝提報者提醒。檢視後，我同意本條目目前最大的問題是獨立二手來源不足，現有內容過度依賴主題自身公開文件，因此未必已達獨立條目的收錄標準。

不過，本頁並非測試頁、空泛短文或純粹惡作劇內容；若以快速刪除處理，可能稍嫌過快。若管理員或其他編者認為現階段不宜保留於條目空間，懇請優先考慮移至草稿頁或使用者沙盒，讓我有機會依照維基百科方針補充獨立來源、重寫語氣並刪去不必要內容後，再重新提交。

--~~~~
```

## 我對目前條目本身的誠實判斷

如果是要「保住不被快速刪除」，妳有可以主張的空間。

如果是要「直接說服社群這篇現在已經足夠當正式條目」，目前其實偏弱，原因是：

1. 幾乎所有來源都來自主題自身文件，這很難用來證明關注度。
2. 條目裡有些內容仍偏體系內自我描述，例如「底層認知共識協議」「開放條款與謙遜條款」等，容易被看成主題自我陳述而不是百科描述。
3. 出版資訊目前看起來仍偏自出版或自家出版脈絡，若沒有外部書目、評論、採訪或新聞報導支撐，單靠這些通常不夠。

---

## 如果還要再補強，最先該刪或改的段落

優先縮掉這些部分：

1. 「文本組成」整段。
2. 「開放條款與謙遜條款」整段。
3. 任何像宣言、定位、價值主張的句子。
4. 任何只能由主題自己聲稱、外部來源無法驗證的描述。

優先保留這些部分：

1. 這是什麼。
2. 何時公開發布。
3. 哪裡可查到公開文本。
4. 是否已有外部媒體、評論、研究或正式出版記錄可佐證。

## 目前適合妳採取的立場

最穩的說法不是：

「這篇一定符合收錄標準，請不要刪。」

而是：

「我承認來源還需要補強，但這不是明顯該快速刪除的頁面；若正式條目空間仍不合適，請先改交存廢討論或移至草稿，讓我補件。」

這樣比較符合維基社群會接受的語氣，也比較不會讓人覺得妳在跟方針硬碰硬。
